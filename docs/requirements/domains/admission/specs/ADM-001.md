## Summary

`submit()` is the primary public entry point for transaction submission into the dig-mempool. It is the function that DIG L2 full nodes, wallets, and peer relay layers call to propose a `SpendBundle` for inclusion in a future block. The function accepts the raw spend bundle, the coin records for all on-chain coins it references, and the current chain height and timestamp.

This function mirrors the role played by Chia L1's `MempoolManager.add_spend_bundle()` ([`mempool_manager.py:538-607`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L538)), which is the entry point for all mempool submissions on the Chia network. On Chia L1, `add_spend_bundle()` delegates to `validate_spend_bundle_and_add_to_pool()` ([`mempool_manager.py:609`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L609)) which orchestrates the full admission pipeline. The dig-mempool consolidates this into a single `submit()` method on the `Mempool` struct.

The function uses `&self` (not `&mut self`) because the `Mempool` uses interior mutability with `RwLock` for fine-grained locking. This allows multiple concurrent submissions to validate in parallel (Phase 1) while only serializing the fast state-mutation step (Phase 2). The caller is responsible for providing coin records -- the mempool never performs I/O or database lookups.

## Specification

### Function Signature

```rust
impl Mempool {
    pub fn submit(
        &self,
        bundle: SpendBundle,
        coin_records: &HashMap<Bytes32, CoinRecord>,
        current_height: u64,
        current_timestamp: u64,
    ) -> Result<SubmitResult, MempoolError>;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `bundle` | `SpendBundle` | The spend bundle to validate and admit. Contains `coin_spends: Vec<CoinSpend>` and `aggregated_signature: Signature`. From `chia-protocol` via `dig-clvm` re-exports. Consumed by value (moved into `MempoolItem` on success). |
| `coin_records` | `&HashMap<Bytes32, CoinRecord>` | Coin state for every on-chain coin spent by the bundle. Keys are coin IDs. The caller must provide these; the mempool does not perform lookups. For CPFP children, use `get_mempool_coin_record()` to obtain synthetic records for mempool-created coins. |
| `current_height` | `u64` | The current L2 block height. Used for timelock resolution and pending determination. |
| `current_timestamp` | `u64` | The current L2 block timestamp. Used for time-based timelock resolution. |

### Return Type

- **`Ok(SubmitResult::Success)`** -- Bundle was admitted to the active mempool.
- **`Ok(SubmitResult::Pending { assert_height })`** -- Bundle is valid but timelocked; stored in the pending pool.
- **`Err(MempoolError)`** -- Admission failed. The error variant indicates the specific failure.

### Concurrency Model

The function takes `&self` because `Mempool` uses interior mutability:
- **Phase 1** (CLVM validation): Runs lock-free. Multiple `submit()` calls validate concurrently.
- **Phase 2** (state mutation): Acquires write lock. Only fast HashMap operations.

## Acceptance Criteria

- [ ] The method `submit()` exists on `Mempool` with the exact signature specified above.
- [ ] The method takes `&self` (not `&mut self`) for interior mutability support.
- [ ] `SpendBundle` is consumed by value (owned, not borrowed).
- [ ] `coin_records` is borrowed as `&HashMap<Bytes32, CoinRecord>`.
- [ ] Returns `Result<SubmitResult, MempoolError>` with correct variants.
- [ ] The method is publicly accessible from external crates depending on `dig-mempool`.
- [ ] A valid spend bundle with correct coin records returns `Ok(SubmitResult::Success)`.
- [ ] A timelocked bundle returns `Ok(SubmitResult::Pending { assert_height })`.

---

## Implementation Notes

- **Primary file:** `src/mempool.rs` (or `src/admission.rs`)
- **Dependencies:** ADM-002 through ADM-007 (submit orchestrates the full pipeline)
- **Edge cases:**
  - Empty `coin_records` with a bundle that spends coins should fail with `CoinNotFound` (from dig-clvm validation).
  - Empty `SpendBundle` (zero coin spends) -- dig-clvm handles this case; the mempool should propagate the result.
  - `current_height = 0` is valid (genesis).
