## Summary

Virtual cost is the mempool's adjusted cost metric that penalizes bundles with many coin spends. Raw CLVM cost alone does not capture the full resource cost of a transaction -- each coin spend requires separate UTXO lookups, state updates, and I/O operations at the database layer. The virtual cost adds a per-spend penalty to account for these externalities, incentivizing bundling multiple operations into fewer spends.

This matches Chia L1's `MempoolItem.virtual_cost` calculation at [`mempool_item.py:92-93`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L92): `virtual_cost = cost + (num_spends * SPEND_PENALTY_COST)`. Chia uses `SPEND_PENALTY_COST = 500_000` ([`mempool_item.py:14`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L14)).

The fee-per-virtual-cost (FPC) is the primary ordering metric for the mempool. All priority decisions (eviction, selection, RBF comparison) use FPC. Chia L1 uses floating-point for FPC ([`mempool_item.py:76-77`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L76)), but dig-mempool uses scaled integers (`FPC_SCALE = 1_000_000_000_000`) to avoid floating-point non-determinism across platforms.

## Specification

### Virtual Cost Formula

```rust
let cost: u64 = spend_result.conditions.cost;
let num_spends: usize = bundle.coin_spends.len();
let virtual_cost: u64 = cost + (num_spends as u64 * config.spend_penalty_cost);
```

Where:
- `cost` = CLVM execution cost from `OwnedSpendBundleConditions.cost` (= `execution_cost + condition_cost`)
- `num_spends` = number of `CoinSpend` entries in the bundle
- `spend_penalty_cost` = `config.spend_penalty_cost` (default: `SPEND_PENALTY_COST = 500_000`)

### Fee-Per-Virtual-Cost (Scaled)

```rust
let fee_per_virtual_cost_scaled: u128 = if virtual_cost == 0 {
    0  // edge case: empty bundle
} else {
    (fee as u128 * FPC_SCALE) / virtual_cost as u128
};
```

Where `FPC_SCALE = 1_000_000_000_000` (10^12).

### Max Bundle Cost Check

```rust
if cost > config.max_bundle_cost {
    return Err(MempoolError::CostExceeded {
        cost,
        max: config.max_bundle_cost,
    });
}
```

Default `max_bundle_cost` = `L1_MAX_COST_PER_SPEND = 11_000_000_000` (from dig-clvm `config.rs:6`), matching Chia L1's [`MAX_BLOCK_COST_CLVM`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/consensus/default_constants.py#L68). Note: this is a per-bundle limit, separate from dig-clvm's per-block validation cost limit.

### Constants

| Constant | Value | Source |
|----------|-------|--------|
| `SPEND_PENALTY_COST` | `500_000` | SPEC Section 2.5, Chia `mempool_item.py:14` |
| `FPC_SCALE` | `1_000_000_000_000` | SPEC Section 2.5 |
| Default `max_bundle_cost` | `11_000_000_000` | dig-clvm `config.rs:6`, Chia `MAX_BLOCK_COST_CLVM` |

### Example

For a bundle with `cost = 5_000_000`, `num_spends = 3`, `fee = 100_000_000`:

```
virtual_cost = 5_000_000 + (3 * 500_000) = 6_500_000
fee_per_virtual_cost_scaled = (100_000_000 * 1_000_000_000_000) / 6_500_000
                            = 15_384_615_384_615
```

## Acceptance Criteria

- [ ] Virtual cost is computed as `cost + (num_spends * SPEND_PENALTY_COST)`.
- [ ] `cost` is read from `SpendResult.conditions.cost`, not recomputed.
- [ ] `num_spends` is `bundle.coin_spends.len()`.
- [ ] Fee-per-virtual-cost uses integer arithmetic with `FPC_SCALE` (no floating point).
- [ ] If `cost > config.max_bundle_cost`, returns `MempoolError::CostExceeded { cost, max }`.
- [ ] The computed `virtual_cost` and `fee_per_virtual_cost_scaled` are stored in the `MempoolItem`.
- [ ] `virtual_cost == 0` edge case is handled without division by zero.

## Implementation Notes

- **Primary file:** `src/admission.rs` (cost computation step)
- **Dependencies:** ADM-004 (provides the fee), ADM-002 (provides the SpendResult with cost)
- **Edge cases:**
  - `virtual_cost = 0`: Only possible with zero cost and zero spends (empty bundle). FPC should be 0.
  - `fee = 0`: FPC = 0. Valid for zero-fee transactions.
  - Overflow: `fee as u128 * FPC_SCALE` can be at most `u64::MAX * 10^12 ≈ 1.8 * 10^31`, which fits in u128 (`max ≈ 3.4 * 10^38`).
  - `cost = max_bundle_cost`: Should pass (not strict inequality).
