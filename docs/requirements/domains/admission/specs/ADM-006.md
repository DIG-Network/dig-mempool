## Summary

Timelock resolution converts relative timelock conditions into absolute values for efficient storage and comparison. CLVM puzzles can assert timelocks using both absolute and relative conditions. Absolute timelocks (e.g., `ASSERT_HEIGHT_ABSOLUTE 1000`) specify a chain state directly. Relative timelocks (e.g., `ASSERT_HEIGHT_RELATIVE 10`) are relative to the coin's confirmed block index -- a coin confirmed at height 500 with `ASSERT_HEIGHT_RELATIVE 10` cannot be spent until height 510.

The mempool resolves all relative conditions to absolute values at admission time, storing only absolute timelocks in the `MempoolItem`. This allows efficient comparison during block selection and expiry checks without re-resolving against coin records.

This mirrors Chia L1's `compute_assert_height()` ([`mempool_manager.py:81-126`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L81)), which resolves `ASSERT_HEIGHT_RELATIVE` and `ASSERT_SECONDS_RELATIVE` using the coin record's `confirmed_block_index` and `timestamp`. The resolution logic handles four condition types per spend, plus bundle-level absolutes.

After resolution, the mempool determines the bundle's status: impossible constraints are rejected, already-expired bundles are rejected, and future-timelocked bundles are routed to the pending pool.

## Specification

### Relative to Absolute Resolution

For each spend `s` in `conditions.spends[]`, using the corresponding `coin_record`:

```rust
// Height relative -> absolute
if let Some(height_relative) = s.height_relative {
    let resolved = coin_record.confirmed_block_index as u64 + height_relative as u64;
    assert_height = max(assert_height, resolved);
}

// Seconds relative -> absolute
if let Some(seconds_relative) = s.seconds_relative {
    let resolved = coin_record.timestamp + seconds_relative;
    assert_seconds = max(assert_seconds, resolved);
}

// Before-height relative -> absolute
if let Some(before_height_relative) = s.before_height_relative {
    let resolved = coin_record.confirmed_block_index as u64 + before_height_relative as u64;
    assert_before_height = min(assert_before_height, resolved);
}

// Before-seconds relative -> absolute
if let Some(before_seconds_relative) = s.before_seconds_relative {
    let resolved = coin_record.timestamp + before_seconds_relative;
    assert_before_seconds = min(assert_before_seconds, resolved);
}
```

### Bundle-Level Absolutes

Bundle-level absolute timelocks from `conditions`:

```rust
if conditions.height_absolute > 0 {
    assert_height = max(assert_height, conditions.height_absolute as u64);
}
if conditions.seconds_absolute > 0 {
    assert_seconds = max(assert_seconds, conditions.seconds_absolute);
}
if let Some(bha) = conditions.before_height_absolute {
    assert_before_height = min(assert_before_height, bha as u64);
}
if let Some(bsa) = conditions.before_seconds_absolute {
    assert_before_seconds = min(assert_before_seconds, bsa);
}
```

### Impossible Constraint Detection

```rust
if let (Some(ah), Some(abh)) = (assert_height, assert_before_height) {
    if abh <= ah {
        return Err(MempoolError::ImpossibleTimelocks);
    }
}
if let (Some(as_), Some(abs)) = (assert_seconds, assert_before_seconds) {
    if abs <= as_ {
        return Err(MempoolError::ImpossibleTimelocks);
    }
}
```

A bundle that requires `assert_height >= 100` AND `assert_before_height <= 100` is impossible -- it can never be valid at any height.

### Expiry Check

```rust
if let Some(abh) = assert_before_height {
    if current_height >= abh {
        return Err(MempoolError::Expired);
    }
}
if let Some(abs) = assert_before_seconds {
    if current_timestamp >= abs {
        return Err(MempoolError::Expired);
    }
}
```

### Pending Determination

```rust
let is_pending = match assert_height {
    Some(ah) if ah > current_height => true,
    _ => false,
} || match assert_seconds {
    Some(as_) if as_ > current_timestamp => true,
    _ => false,
};

if is_pending {
    // Route to pending pool
    return Ok(SubmitResult::Pending { assert_height: assert_height.unwrap_or(0) });
}
```

### CPFP Mempool Coin Handling

For coins that are not in `coin_records` (CPFP mempool coin candidates), use:
- `confirmed_block_index = current_height`
- `timestamp = current_timestamp`

This matches Chia's ephemeral coin handling at [`mempool_manager.py:716-722`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L716).

## Acceptance Criteria

- [ ] `height_relative` is resolved to `coin_record.confirmed_block_index + n`.
- [ ] `seconds_relative` is resolved to `coin_record.timestamp + n`.
- [ ] `before_height_relative` is resolved and min'd with existing `assert_before_height`.
- [ ] `before_seconds_relative` is resolved and min'd with existing `assert_before_seconds`.
- [ ] Bundle-level absolutes are merged with per-spend resolved values (max for asserts, min for befores).
- [ ] Impossible constraints (`assert_before_height <= assert_height`) return `MempoolError::ImpossibleTimelocks`.
- [ ] Already-expired bundles return `MempoolError::Expired`.
- [ ] Future-timelocked bundles return `Ok(SubmitResult::Pending { assert_height })`.
- [ ] Resolved absolute values are stored in `MempoolItem` (`assert_height`, `assert_before_height`, `assert_before_seconds`).

## Implementation Notes

- **Primary file:** `src/admission.rs` (timelock resolution step)
- **Dependencies:** ADM-002 (provides conditions with per-spend timelocks), ADM-004 (provides coin records context)
- **Edge cases:**
  - No timelocks: All `assert_*` fields are `None`. Bundle goes to active pool.
  - Only expiry conditions (before_height only): `assert_height = None`, `assert_before_height = Some(X)`. Active if not expired.
  - Relative timelock on genesis coin (`confirmed_block_index = 0`): resolved to `0 + relative = relative`.
  - Multiple spends with different timelocks: `assert_height = max(all resolved)`, `assert_before_height = min(all resolved)`.
  - CPFP child coin with relative timelock: resolved against `current_height` / `current_timestamp` (may become pending).
