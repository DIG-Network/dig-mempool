# POL-002: Active Pool Capacity Management

> **Authoritative requirement:** [POL-002](../NORMATIVE.md#POL-002)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The active pool has a maximum aggregate cost limit (`max_total_cost`). When admitting a new item would exceed this limit, the mempool evicts lowest-value items to make room. Items are evicted in ascending `descendant_score` order -- this protects low-fee parents that have valuable children (CPFP). If the new item's fee-per-cost is lower than or equal to the lowest `descendant_score` in the pool, the item is rejected outright with `MempoolFull`.

This corresponds to Chia's `add_to_pool()` ([`mempool.py:395-502`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L395)), which evicts items sorted by `fee_per_cost` ascending ([`mempool.py:448-458`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L448)). dig-mempool improves on this by using `descendant_score` instead of raw `fee_per_cost`, which accounts for CPFP child value.

## Specification

### Eviction Trigger

Eviction is triggered when:

```
total_cost + new_item.virtual_cost > max_total_cost
```

Where `total_cost` is the current aggregate `virtual_cost` of all active pool items, and `max_total_cost` is configured via `MempoolConfig.max_total_cost` (default: `L2_MAX_COST_PER_BLOCK * MEMPOOL_BLOCK_BUFFER` = `550B * 15` = `8.25T`).

### Eviction Algorithm

1. Sort all active items by `descendant_score` ascending (lowest value first).
2. Skip items protected by expiry ([POL-003](POL-003.md)).
3. For each candidate in ascending score order:
   a. If the new item's `fee_per_virtual_cost_scaled <= candidate.descendant_score`, reject the new item with `MempoolFull` (it cannot beat any remaining item).
   b. Remove the candidate and all its dependents (cascade eviction). Fire `on_item_removed(CapacityEviction)`.
   c. Recalculate `total_cost`. If sufficient space is now available, stop evicting.
4. If sufficient space cannot be freed (all remaining items are protected or higher-value), reject with `MempoolFull`.

### Spend Count Limit

In addition to cost limits, the active pool enforces `config.max_spends_per_block`:

```
total_spends + new_item.num_spends > max_spends_per_block  ->  TooManySpends
```

### Cascade Eviction

When an item is evicted, all its dependents (items whose `depends_on` includes the evicted item) are also removed recursively. Cascade-evicted items do NOT go to the conflict cache.

### Descendant Score

`descendant_score` is the maximum of:
- The item's own `fee_per_virtual_cost_scaled`.
- The `package_fee_per_virtual_cost_scaled` of any descendant chain.

This ensures a low-fee parent with a high-fee child is protected from eviction.

---

## Acceptance Criteria

- [ ] Eviction is triggered when `total_cost + new_item.virtual_cost > max_total_cost`.
- [ ] Items are evicted in ascending `descendant_score` order.
- [ ] Expiry-protected items are skipped during eviction.
- [ ] If the new item's FPC <= lowest evictable `descendant_score`, the item is rejected with `MempoolFull`.
- [ ] Dependents of evicted items are cascade-evicted.
- [ ] Cascade-evicted items do not go to the conflict cache.
- [ ] The `TooManySpends` error is returned when spend count limit is exceeded.

## Implementation Notes

- **Primary file:** `src/pools/active.rs` or `src/admission/capacity.rs`
- **Chia comparison:** Chia sorts by `fee_per_cost` ASC via SQL `ORDER BY fee_per_cost ASC` ([`mempool.py:448`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L448)). dig-mempool uses `descendant_score` for CPFP awareness.
- **Performance:** Sorting all items for eviction is O(n log n). For large mempools, a min-heap on `descendant_score` could provide O(log n) eviction.
- **Ancestor score update:** After evicting dependents, walk up the ancestor chain and recompute `descendant_score` for affected ancestors.

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Eviction triggered at capacity | Unit test | Fill pool to max_total_cost, add one more | Lowest-score item evicted |
| Lowest descendant_score evicted first | Unit test | Items with scores [10, 50, 100], add new item | Score=10 item evicted first |
| MempoolFull when FPC too low | Unit test | New item FPC < all existing descendant_scores | Rejected with MempoolFull |
| Cascade eviction | Unit test | Parent evicted, child depends on parent | Both parent and child removed |
| Space freed stops eviction | Unit test | Evict one item, enough space freed | Only one item evicted |
| Spend count limit | Unit test | total_spends + new > max_spends_per_block | TooManySpends error |
| Expiry-protected skipped | Unit test | Lowest-score item is expiry-protected | Skipped, next item evicted |
| Descendant score protects parent | Unit test | Low-fee parent has high-fee child | Parent's descendant_score reflects child; not evicted before standalone items |
