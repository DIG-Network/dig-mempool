# POL-003: Expiry Protection

> **Authoritative requirement:** [POL-003](../NORMATIVE.md#POL-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Items that are close to their expiry height are protected from capacity-based eviction. An item is considered "expiry-protected" when its `assert_before_height` is within `expiry_protection_blocks` of the current height. This ensures that time-sensitive transactions (e.g., atomic swaps with deadlines) are not evicted moments before they would have been included in a block.

This corresponds to Chia's expiring-item-specific eviction logic at [`mempool.py:406-442`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L406), which uses 48 blocks / 900 seconds as the protection window. dig-mempool uses a configurable `expiry_protection_blocks` (default: 100 blocks) from `MempoolConfig`.

---

## Specification

### Protection Condition

An item is **expiry-protected** if:

```
item.assert_before_height.is_some()
    && item.assert_before_height.unwrap() > current_height
    && item.assert_before_height.unwrap() <= current_height + config.expiry_protection_blocks
```

In words: the item has an expiry height, has not yet expired, and will expire within `expiry_protection_blocks` blocks.

### Eviction Behavior

During capacity-based eviction ([POL-002](POL-002.md)):

1. **Non-expiring items cannot evict expiry-protected items.** When iterating candidates in ascending `descendant_score` order, expiry-protected items are skipped entirely.

2. **Expiring items CAN evict other expiring items** if the new expiring item has a higher `fee_per_virtual_cost_scaled` than the victim. This matches Chia's logic at [`mempool.py:416-442`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L416) where an incoming expiring item can replace a lower-fee expiring item.

3. **If a new item is expiry-protected and the pool is full**, it may evict non-protected items normally (via descendant_score ordering).

### Configuration

| Parameter | Default | Description |
|-----------|---------|-------------|
| `expiry_protection_blocks` | 100 | Blocks before `assert_before_height` within which protection applies |

Chia uses 48 blocks / 900 seconds ([`mempool.py:408-409`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L408)). dig-mempool defaults to 100 blocks, configurable via `MempoolConfig`.

---

## Acceptance Criteria

- [ ] Items within `expiry_protection_blocks` of their `assert_before_height` are skipped during non-expiring eviction.
- [ ] Items without `assert_before_height` are never expiry-protected.
- [ ] Items already expired (`assert_before_height <= current_height`) are not protected.
- [ ] Expiring items can evict other expiring items with lower FPC.
- [ ] The protection window is configurable via `MempoolConfig.expiry_protection_blocks`.
- [ ] Protection applies only during capacity eviction, not during `on_new_block()` expiry removal.

---

## Implementation Notes

- **Primary file:** `src/pools/active.rs` or `src/admission/capacity.rs`
- **current_height:** The height used is the `current_height` passed to `submit()`, not the last `on_new_block()` height.
- **Chia comparison:** Chia has a two-pass eviction: first pass skips expiring items, second pass handles expiring-vs-expiring. dig-mempool can implement this as a single pass with a skip flag.
- **Edge case:** If all low-score items are expiry-protected, eviction fails and the new item is rejected with `MempoolFull`.

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Protected item skipped | Unit test | Item with `assert_before_height = height + 50`, protection = 100 blocks | Skipped during eviction |
| Unprotected item evicted | Unit test | Item with `assert_before_height = height + 200`, protection = 100 blocks | Not protected, evictable |
| No expiry, no protection | Unit test | Item with `assert_before_height = None` | Not protected |
| Already expired, not protected | Unit test | Item with `assert_before_height = height - 5` | Not protected (already expired) |
| Expiring evicts lower-FPC expiring | Unit test | New expiring item FPC=100 vs existing expiring item FPC=50 | Existing evicted |
| MempoolFull when all protected | Unit test | All evictable items are expiry-protected | MempoolFull returned |
| Configurable window | Unit test | Set `expiry_protection_blocks = 50`, item at height + 60 | Not protected (60 > 50) |
| Boundary: exactly at window edge | Unit test | Item at `assert_before_height = height + expiry_protection_blocks` | Protected (within window) |
