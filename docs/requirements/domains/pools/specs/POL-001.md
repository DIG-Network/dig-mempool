## Summary

The active pool is the primary storage for validated, admitted transactions that are eligible for block inclusion. It uses a `HashMap<Bytes32, Arc<MempoolItem>>` keyed by bundle ID (`spend_bundle_id`) for O(1) lookup, insertion, and removal. A secondary `coin_index: HashMap<Bytes32, Bytes32>` maps each spent coin ID to the bundle that spends it, enabling O(1) conflict detection during admission.

This corresponds to Chia's `_items` dict ([`mempool.py:151`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L151)) for the items HashMap, and the `spends` SQLite table ([`mempool.py:146-149`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L146)) for the coin_index. dig-mempool replaces Chia's SQLite in-memory tables ([`mempool.py:108`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L108)) with pure HashMaps for lower overhead.

An additional `mempool_coins: HashMap<Bytes32, Bytes32>` maps coin IDs created by mempool items to their creating bundle ID, enabling CPFP dependency resolution. This is a dig-mempool extension not present in Chia L1.

## Specification

### Data Structures

```rust
// Active pool storage (protected by pool_lock: RwLock)
items:         HashMap<Bytes32, Arc<MempoolItem>>  // bundle_id -> item
coin_index:    HashMap<Bytes32, Bytes32>           // spent_coin_id -> spending bundle_id
mempool_coins: HashMap<Bytes32, Bytes32>           // created_coin_id -> creating bundle_id
```

### Invariants

- Every key in `items` corresponds to a unique `spend_bundle_id`.
- Every coin in `item.removals` has an entry in `coin_index` pointing back to the item's bundle ID.
- Every coin in `item.additions` has an entry in `mempool_coins` pointing back to the item's bundle ID.
- No two items in `items` may spend the same coin (conflict-free invariant enforced by admission pipeline).
- Items are stored as `Arc<MempoolItem>` for zero-copy sharing between reads and the selection algorithm.

### Operations

| Operation | Complexity | Description |
|-----------|-----------|-------------|
| `get(bundle_id)` | O(1) | Look up item by bundle ID |
| `contains(bundle_id)` | O(1) | Check existence |
| `insert(item)` | O(r + a) | Add to items, coin_index (r removals), mempool_coins (a additions) |
| `remove(bundle_id)` | O(r + a) | Remove from items, coin_index, mempool_coins |
| `conflict_check(coin_id)` | O(1) | Look up coin_index for existing spender |
| `cpfp_lookup(coin_id)` | O(1) | Look up mempool_coins for creating bundle |

### Accumulators

The active pool tracks aggregate values updated on every insert/remove:

- `total_cost: u64` -- sum of all items' `virtual_cost`.
- `total_fees: u64` -- sum of all items' `fee`.
- `total_spends: usize` -- sum of all items' `num_spends`.

## Acceptance Criteria

- [ ] `items` is a `HashMap<Bytes32, Arc<MempoolItem>>`.
- [ ] `coin_index` is a `HashMap<Bytes32, Bytes32>` mapping spent coin IDs to bundle IDs.
- [ ] `mempool_coins` is a `HashMap<Bytes32, Bytes32>` mapping created coin IDs to bundle IDs.
- [ ] Insert updates all three maps and accumulators.
- [ ] Remove cleans up all three maps and accumulators.
- [ ] No two items in the active pool spend the same coin.
- [ ] Items are stored as `Arc<MempoolItem>`.

## Implementation Notes

- **Primary file:** `src/pools/active.rs` or `src/mempool.rs`
- **Chia comparison:** Chia's `tx` table (SQL) stores `(name, cost, fee, assert_height, assert_before_height, assert_before_seconds, fee_per_cost)` for query-based selection. dig-mempool stores the full `MempoolItem` in-memory. Chia's `spends` table maps `(coin_id, tx_name)` -- equivalent to `coin_index`.
- **mempool_coins:** Not present in Chia. Required for CPFP dependency resolution (Phase 2 step 8).
- **Arc immutability:** Once an `Arc<MempoolItem>` is created, the item is never mutated. `descendant_score` is the one exception and uses interior mutability (atomic or cell).

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Insert and get | Unit test | Insert item, retrieve by bundle_id | Same item returned |
| coin_index populated | Unit test | Insert item with removals [A, B], check coin_index[A] and coin_index[B] | Both map to bundle_id |
| mempool_coins populated | Unit test | Insert item with additions [X, Y], check mempool_coins[X] | Maps to bundle_id |
| Remove cleans all maps | Unit test | Insert then remove, check all three maps | All entries removed |
| Accumulators update | Unit test | Insert two items, check total_cost = sum | Correct sum |
| Conflict-free invariant | Unit test | Try to insert item spending already-spent coin | Rejected during admission |
| Arc sharing | Unit test | Get item, hold reference, remove original | Arc reference still valid |
| Empty pool operations | Unit test | Get/contains on empty pool | Returns None/false |
