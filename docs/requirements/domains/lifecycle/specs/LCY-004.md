# LCY-004: clear() — Reorg Recovery Reset

> **Authoritative requirement:** [LCY-004](../NORMATIVE.md#LCY-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 3.5, 9.3

## Summary

`clear()` performs a complete reset of all mempool state for use during reorg recovery. When the chain reorganizes to a different fork, the entire mempool may be invalid (coins may not exist, timelocks may differ). The safest recovery strategy is to clear all state and let the caller re-fetch chain state and re-submit known-good bundles.

Event hooks and configuration are preserved across `clear()` so the caller does not need to re-register hooks or reconfigure the mempool after recovery.

---

## Specification

### Function Signature

```rust
impl Mempool {
    pub fn clear(&self);
}
```

### State Cleared

| State | Cleared | Notes |
|-------|---------|-------|
| Active pool (`items`, `coin_index`, `mempool_coins`) | Yes | All active items removed |
| Dependency graph (`dependencies`, `dependents`) | Yes | All relationships removed |
| Singleton tracking (`singleton_spends`) | Yes | All chain state removed |
| Dedup index (`dedup_index`) | Yes | All dedup entries removed |
| Pending pool (`pending`, `pending_coin_index`) | Yes | All pending items removed |
| Conflict cache (`conflict_cache`) | Yes | All conflict items removed |
| Seen cache | Yes | Allows resubmission of previously seen bundles |
| Cost accumulators | Yes | Reset to zero |
| Fee estimator | Yes | Historical fee data cleared |
| Internal height/timestamp | Yes | Reset to zero |

### State Preserved

| State | Preserved | Notes |
|-------|-----------|-------|
| Event hooks (`hooks`) | Yes | Caller does not re-register |
| Configuration (`MempoolConfig`) | Yes | Immutable after construction |
| Network constants | Yes | Immutable after construction |
| BLS cache | Yes | Cache entries are still valid across reorgs |

### Event Hooks

`clear()` MUST fire `on_item_removed(bundle_id, RemovalReason::Cleared)` for every active and pending item before clearing. This allows hooks to perform cleanup (e.g., notify connected wallets).

## Acceptance Criteria

- [ ] `clear()` is publicly exported on the `Mempool` struct.
- [ ] All active pool state is cleared (items, coin_index, mempool_coins).
- [ ] Dependency graph is cleared.
- [ ] Pending pool is cleared.
- [ ] Conflict cache is cleared.
- [ ] Seen cache is cleared.
- [ ] Fee estimator state is cleared.
- [ ] Event hooks are preserved.
- [ ] Configuration is preserved.
- [ ] `on_item_removed(Cleared)` fired for all removed items.
- [ ] After `clear()`, `len()` returns 0, `is_empty()` returns true.

---

## Implementation Notes

- **Primary file:** `src/mempool.rs` or `src/lifecycle.rs`
- **Locking:** Acquires all write locks. Acceptable since reorg recovery is infrequent.
- **Hook firing order:** Fire removal hooks before clearing the data structures (hooks may read item data).
- **Edge cases:**
  - `clear()` on an already-empty mempool: no-op, no hooks fired.
  - `clear()` during concurrent `submit()`: Phase 1 (CLVM validation) may complete after clear. Phase 2 will fail dedup re-check or find the pool empty. This is safe.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Empty after clear | Unit test | Populate mempool, clear(), check len() | len() == 0, is_empty() == true |
| Active items removed | Unit test | Submit items, clear(), get() returns None | All items gone |
| Pending items removed | Unit test | Submit timelocked items, clear() | pending_len() == 0 |
| Conflict cache cleared | Unit test | Create conflicts, clear() | conflict_len() == 0 |
| Seen cache cleared | Unit test | Submit bundle (seen), clear(), resubmit | Not rejected as AlreadySeen |
| Hooks preserved | Unit test | Register hook, clear(), verify hook still registered | Hook still receives future events |
| Config preserved | Unit test | Clear, submit new bundle | Config (max_total_cost etc.) unchanged |
| Removal hooks fired | Unit test | Register hook, populate, clear() | on_item_removed(Cleared) called for each item |
