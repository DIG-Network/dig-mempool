# LCY-007: snapshot() / restore() — Persistence

> **Authoritative requirement:** [LCY-007](../NORMATIVE.md#LCY-007)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 3.9

## Summary

`snapshot()` and `restore()` provide mempool persistence across process restarts. The mempool serializes its complete state into a `MempoolSnapshot` struct that derives `serde::Serialize` and `serde::Deserialize`. The caller is responsible for actual I/O (writing to disk, sending over network, etc.), maintaining the "no I/O" principle.

This is a dig-mempool extension not present in Chia L1 (SPEC Section 14.2). Chia's mempool is rebuilt from scratch on every restart.

The snapshot includes active items, pending items, conflict cache bundles, and fee estimator state. It deliberately excludes the seen-cache (ephemeral, DoS protection only) and BLS cache (can be rebuilt from validation).

## Specification

### Function Signatures

```rust
impl Mempool {
    pub fn snapshot(&self) -> MempoolSnapshot;
    pub fn restore(&self, snapshot: MempoolSnapshot);
}
```

### MempoolSnapshot Struct

```rust
#[derive(Serialize, Deserialize)]
pub struct MempoolSnapshot {
    pub active_items: Vec<MempoolItem>,
    pub pending_items: Vec<MempoolItem>,
    pub conflict_bundles: Vec<SpendBundle>,
    pub fee_estimator_state: FeeEstimatorState,
    pub height: u64,
    pub timestamp: u64,
}
```

### Included State

| State | Included | Serialization |
|-------|----------|---------------|
| Active pool items | Yes | `Vec<MempoolItem>` |
| Pending pool items | Yes | `Vec<MempoolItem>` |
| Conflict cache bundles | Yes | `Vec<SpendBundle>` (raw bundles, not validated) |
| Fee estimator state | Yes | `FeeEstimatorState` (see [FEE-005](../../fee_estimation/specs/FEE-005.md)) |
| Current height | Yes | `u64` |
| Current timestamp | Yes | `u64` |

### Excluded State (Rebuilt on Restore)

| State | Excluded | Reason |
|-------|----------|--------|
| Seen cache | Yes | Ephemeral DoS protection; stale after restart |
| BLS cache | Yes | Rebuilt during validation; not serializable |
| `coin_index` | Yes | Rebuilt from active items' removals |
| `mempool_coins` | Yes | Rebuilt from active items' additions |
| Dependency graph | Yes | Rebuilt from active items' `depends_on` |
| `singleton_spends` | Yes | Rebuilt from active items' `singleton_lineage` |
| `dedup_index` | Yes | Rebuilt from active items' dedup data |

### snapshot() Behavior

1. Acquire read locks on all pools.
2. Clone all active items (without `Arc` wrapper).
3. Clone all pending items.
4. Clone all conflict cache bundles.
5. Extract fee estimator state via `FeeTracker::to_state()`.
6. Record current height and timestamp.
7. Return `MempoolSnapshot`.

### restore() Behavior

1. Acquire write locks on all pools.
2. Clear all current state (equivalent to `clear()` but without firing hooks).
3. Rebuild active pool: insert items, rebuild `coin_index`, `mempool_coins`, dependency graph, `singleton_spends`, `dedup_index`.
4. Rebuild pending pool: insert items, rebuild `pending_coin_index`.
5. Restore conflict cache.
6. Restore fee estimator from `FeeEstimatorState`.
7. Set height and timestamp.
8. **Do NOT re-validate items.** The caller should re-validate against current chain state if the snapshot may be stale.

## Acceptance Criteria

- [ ] `snapshot()` is publicly exported on `Mempool` and returns `MempoolSnapshot`.
- [ ] `restore()` is publicly exported on `Mempool` and accepts `MempoolSnapshot`.
- [ ] `MempoolSnapshot` derives `serde::Serialize + serde::Deserialize`.
- [ ] Snapshot includes active items, pending items, conflict bundles, fee estimator state, height, timestamp.
- [ ] Snapshot does NOT include seen-cache or BLS cache.
- [ ] `restore()` rebuilds all derived indexes (coin_index, mempool_coins, dependency graph).
- [ ] Round-trip: `restore(snapshot())` produces equivalent mempool state.
- [ ] `restore()` does not re-validate items (caller responsibility).

## Implementation Notes

- **Primary file:** `src/persistence.rs` or `src/mempool.rs`
- **MempoolItem serialization:** `MempoolItem` contains `SpendBundle`, `Coin`, `Bytes32`, `OwnedSpendBundleConditions`, etc. These types from `chia-protocol` and `chia-consensus` implement `Streamable` (chia-traits). serde support may require wrapper types or manual implementations for types that don't derive serde.
- **Performance target:** snapshot < 50ms, restore < 100ms (SPEC Section 17).
- **Edge cases:**
  - Restoring a snapshot from a different config (e.g., different max_total_cost): items may exceed new limits. The caller should handle this.
  - Restoring a snapshot from a future height: the mempool accepts it. The caller is responsible for consistency.
  - Empty snapshot: valid, produces an empty mempool.

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Round-trip preserves state | Integration test | Populate pool, snapshot, clear, restore | len(), stats() match original |
| Active items preserved | Integration test | Submit items, snapshot, restore | get() returns same items |
| Pending items preserved | Integration test | Submit timelocked items, snapshot, restore | pending_len() matches |
| Conflict cache preserved | Integration test | Create conflicts, snapshot, restore | conflict_len() matches |
| Fee estimator preserved | Integration test | Record blocks, snapshot, restore | estimate_fee_rate() returns same values |
