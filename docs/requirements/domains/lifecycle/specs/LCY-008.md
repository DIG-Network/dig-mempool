## Summary

`evict_lowest_percent(percent)` is an emergency pressure-relief mechanism that removes the lowest-value items from the active pool. It is designed for external memory pressure handlers to call when the system is under resource constraints.

Items are ordered by `descendant_score` ascending (lowest value first). The `descendant_score` is the maximum of the item's own `fee_per_virtual_cost_scaled` and the `package_fee_per_virtual_cost_scaled` of any descendant chain. This protects low-fee parents that have valuable children (CPFP protection).

Items within the expiry protection window are skipped to avoid evicting transactions that are about to expire naturally (they will be cleaned up by `on_new_block()` soon). When evicting an item that has dependents, those dependents are cascade-evicted as well.

The function corresponds to Chia's capacity management in `add_to_pool()` ([`mempool.py:446-460`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L446)) but is exposed as a standalone operation for external pressure management, with the recommended thresholds in SPEC Section 9.4: 25% at warning severity, 50% at critical severity.

## Specification

### Function Signature

```rust
impl Mempool {
    pub fn evict_lowest_percent(&self, percent: u8);
}
```

### Algorithm

1. Compute target cost to remove: `target = total_cost * percent / 100`.
2. Sort all active items by `descendant_score` ascending.
3. Iterate sorted items:
   a. Skip items within expiry protection window (`assert_before_height - current_height <= config.expiry_protection_blocks`). Chia: 48 blocks / 900 seconds ([`mempool.py:406-442`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L406)).
   b. Remove item from active pool.
   c. Cascade-evict all dependents of the removed item.
   d. Fire `on_item_removed(CapacityEviction)` for the primary item.
   e. Fire `on_item_removed(CascadeEvicted)` for each cascaded dependent.
   f. Add removed item's cost (and cascaded dependents' costs) to running total.
   g. Stop when running total >= target.
4. Recompute `descendant_score` for ancestors of removed items.
5. Update cost accumulators.

### Expiry Protection

Items are protected from eviction if:
- They have `assert_before_height` set, AND
- `assert_before_height - current_height <= config.expiry_protection_blocks` (default: 100 blocks)

This prevents evicting items that the caller explicitly time-bounded and that will expire naturally soon.

### Cascade Eviction

When a parent item is evicted, its dependents become invalid (their input coins no longer exist in the mempool). They are removed recursively via the `dependents` graph, using the `CascadeEvicted` removal reason.

## Acceptance Criteria

- [ ] `evict_lowest_percent()` is publicly exported on the `Mempool` struct.
- [ ] Accepts `percent: u8` (0-100).
- [ ] Evicts items sorted by `descendant_score` ascending (lowest value first).
- [ ] Respects expiry protection window.
- [ ] Cascade-evicts dependents of removed items.
- [ ] Fires `on_item_removed` with `CapacityEviction` for primary evictions.
- [ ] Fires `on_item_removed` with `CascadeEvicted` for cascade-evicted dependents.
- [ ] Stops when target cost reduction is reached.
- [ ] `percent = 0` is a no-op.
- [ ] `percent = 100` evicts everything (except expiry-protected items).

## Implementation Notes

- **Primary file:** `src/eviction.rs` or `src/mempool.rs`
- **Locking:** Acquires write lock on pool.
- **Sort cost:** Sorting all items by `descendant_score` is O(n log n). For large pools, consider maintaining a sorted index.
- **Descendant score recomputation:** After removing items, walk up ancestor chains and recompute `descendant_score` as `max(own_fpc, max(descendant.package_fpc))`.
- **Edge cases:**
  - `percent > 100`: treated as 100.
  - All items are expiry-protected: nothing evicted.
  - Cascade eviction removes more cost than target: acceptable (don't partially evict a chain).
