## Summary

`RetryBundles` is the return type of `on_new_block()`. It provides the caller with bundles that should be resubmitted to the mempool after a block confirmation, plus informational data about cascade-evicted items.

On Chia L1, drained pending and conflict items are re-validated inline under the lock ([`mempool_manager.py:900-918`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L900)). dig-mempool returns raw `SpendBundle`s for the caller to resubmit with fresh `coin_records`, maintaining the "no I/O" principle (the mempool does not look up current coin state).

The `cascade_evicted` field contains bundle IDs of items that were removed because their parent was confirmed or expired. These cannot be retried (their input coins no longer exist in the mempool) and are provided for caller bookkeeping, logging, and notification purposes.

## Specification

### Struct Definition

```rust
pub struct RetryBundles {
    /// Bundles from the conflict cache whose conflicting item was removed.
    pub conflict_retries: Vec<SpendBundle>,
    /// Bundles from the pending pool whose timelocks are now satisfied.
    pub pending_promotions: Vec<SpendBundle>,
    /// Bundle IDs that were cascade-evicted because their parent was
    /// confirmed or expired. These cannot be retried (their input coins
    /// no longer exist in the mempool). Provided for caller bookkeeping.
    pub cascade_evicted: Vec<Bytes32>,
}
```

### Field Semantics

| Field | Source | Action Required |
|-------|--------|-----------------|
| `conflict_retries` | Conflict cache | Caller SHOULD resubmit via `submit()` with current coin records |
| `pending_promotions` | Pending pool | Caller SHOULD resubmit via `submit()` with current coin records |
| `cascade_evicted` | Active pool (dependents) | Informational only; caller MAY log or notify users |

### Guarantees

- All items in `conflict_retries` have been removed from the conflict cache.
- All items in `pending_promotions` have been removed from the pending pool.
- All IDs in `cascade_evicted` have been removed from the active pool.
- The struct is always returned, even if all fields are empty vectors.

## Implementation Notes

- **Primary file:** `src/types.rs` or `src/lifecycle.rs`
- **Ownership:** `SpendBundle` is moved out of the internal caches into the return value. The mempool no longer holds references to these bundles.
- **Ordering:** No specific ordering is required for any of the vectors.
- **Edge cases:**
  - No conflicts resolved: `conflict_retries` is empty.
  - No pending promotions: `pending_promotions` is empty.
  - No cascade evictions: `cascade_evicted` is empty.
  - All fields empty: valid return value (no-op block).
