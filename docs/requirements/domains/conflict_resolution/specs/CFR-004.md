# CFR-004: RBF Minimum Fee Bump

> **Authoritative requirement:** [CFR-004](../NORMATIVE.md#CFR-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 5.11

---

## Summary

The minimum fee bump is the third and final RBF condition. After verifying the superset rule (CFR-002) and fee-per-virtual-cost comparison (CFR-003), the mempool must confirm that the replacement bundle's absolute fee exceeds the sum of all conflicting bundles' fees by at least `MIN_RBF_FEE_BUMP` (10,000,000 mojos by default).

This absolute minimum prevents "penny-shaving" attacks where an attacker repeatedly replaces a transaction with marginally higher fees, forcing the mempool to process many replacements for negligible economic improvement. By requiring a meaningful absolute increase, the mempool ensures that each replacement provides a substantial fee improvement, making DoS via repeated replacement economically unviable.

On Chia L1, this is enforced via the `MEMPOOL_MIN_FEE_INCREASE` constant (10,000,000 mojos), checked in `can_replace()`. dig-mempool matches this constant as `MIN_RBF_FEE_BUMP` and makes it configurable via `MempoolConfig.min_rbf_fee_bump`.

---

## Specification

### Constants

```rust
/// Chia: MEMPOOL_MIN_FEE_INCREASE = 10000000 (mempool_manager.py:52)
pub const MIN_RBF_FEE_BUMP: u64 = 10_000_000;
```

### Algorithm

Given:
- `new_fee`: the new bundle's fee.
- `conflicting_items`: items identified by CFR-001.
- `min_rbf_fee_bump`: from `MempoolConfig` (default `MIN_RBF_FEE_BUMP`).

Compute:
```
conflicting_fees = sum(item.fee for item in conflicting_items)
required_fee = conflicting_fees + min_rbf_fee_bump
```

Check:
```
if new_fee < required_fee {
    return Err(MempoolError::RbfBumpTooLow {
        required: required_fee,
        provided: new_fee,
    });
}
```

### Error Type

```rust
MempoolError::RbfBumpTooLow { required: u64, provided: u64 }
```

The error includes both the required and provided fee values for diagnostic purposes.

---

## Acceptance Criteria

- [ ] The new bundle's fee is compared against `conflicting_fees + min_rbf_fee_bump`.
- [ ] The default `min_rbf_fee_bump` is 10,000,000 mojos (10M).
- [ ] `min_rbf_fee_bump` is configurable via `MempoolConfig`.
- [ ] `MempoolError::RbfBumpTooLow` is returned with both `required` and `provided` values when the check fails.
- [ ] The fee comparison is against the **sum** of all conflicting bundles' fees (not individual comparisons).
- [ ] Overflow is handled safely when computing `conflicting_fees + min_rbf_fee_bump`.

---

## Implementation Notes

- **Primary file:** `src/mempool/conflict.rs`
- **Constant definition:** `src/mempool/constants.rs` or `src/constants.rs`
- **Configurable:** `MempoolConfig.min_rbf_fee_bump` defaults to `MIN_RBF_FEE_BUMP` (10M mojos).
- **Overflow protection:** `conflicting_fees + min_rbf_fee_bump` should use `checked_add()` or `saturating_add()`. In practice, fees fit comfortably in `u64` (max ~18.4 exajoules worth of mojos), but defensive coding is appropriate.
- **Ordering of checks:** This is the final RBF check. If CFR-002 (superset) or CFR-003 (FPC) fails first, this check is never reached.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Exact minimum bump passes | Unit test | New fee = conflicting_fees + MIN_RBF_FEE_BUMP exactly | Check passes (>= comparison) |
| Above minimum bump passes | Unit test | New fee = conflicting_fees + 2 * MIN_RBF_FEE_BUMP | Check passes |
| Below minimum bump fails | Unit test | New fee = conflicting_fees + MIN_RBF_FEE_BUMP - 1 | `RbfBumpTooLow` error with correct required/provided |
| Multiple conflicts aggregated | Unit test | Two conflicting bundles with fees 100M and 200M each; new fee must be >= 310M | Minimum required = 300M + 10M = 310M |
| Custom config value | Unit test | Set min_rbf_fee_bump to 50M; verify new threshold | Requires conflicting_fees + 50M |
| Zero-fee conflicting | Unit test | Conflicting bundle has 0 fee; new bundle fee = MIN_RBF_FEE_BUMP | Check passes |
| Error includes both values | Unit test | Trigger RbfBumpTooLow; inspect required and provided fields | Both fields match expected values |
