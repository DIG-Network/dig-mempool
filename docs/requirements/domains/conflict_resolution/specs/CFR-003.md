# CFR-003: RBF Fee-Per-Virtual-Cost Must Be Strictly Higher

> **Authoritative requirement:** [CFR-003](../NORMATIVE.md#CFR-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 5.11

---

## Summary

The fee-per-virtual-cost (FPC) comparison is the second of three RBF conditions. After verifying the superset rule (CFR-002), the mempool must confirm that the replacement bundle offers a strictly higher fee rate than the aggregate fee rate of all conflicting bundles combined. This ensures that replacement transactions always improve the mempool's economic quality, preventing abuse where a replacement merely shuffles coins without increasing miner revenue.

On Chia L1, `can_replace()` compares the new item's `fee_per_cost` against the ratio `conflicting_fees / conflicting_cost`. Chia uses floating-point for this comparison. dig-mempool uses scaled integers (`fee_per_virtual_cost_scaled` with `FPC_SCALE = 1_000_000_000_000`) to avoid non-deterministic floating-point behavior.

The comparison is against the **aggregate** fee rate of all conflicting bundles, not individual comparisons. This means `new_fpc > total_conflicting_fees / total_conflicting_cost`, computed using integer arithmetic to avoid precision loss.

---

## Specification

### Algorithm

Given:
- `new_item_fpc_scaled`: the new bundle's `fee_per_virtual_cost_scaled`.
- `conflicting_items`: items identified by CFR-001.

Compute:
```
conflicting_fees = sum(item.fee for item in conflicting_items)
conflicting_cost = sum(item.cost for item in conflicting_items)
conflicting_fpc_scaled = (conflicting_fees * FPC_SCALE) / conflicting_cost
```

Check:
```
if new_item_fpc_scaled <= conflicting_fpc_scaled {
    return Err(MempoolError::RbfFpcNotHigher);
}
```

The comparison MUST be **strictly greater than** (not greater-than-or-equal).

### Integer Arithmetic

All FPC comparisons use `u128` scaled values to avoid floating-point non-determinism:
```rust
pub const FPC_SCALE: u128 = 1_000_000_000_000;
fee_per_virtual_cost_scaled = (fee as u128 * FPC_SCALE) / virtual_cost as u128;
```

---

## Acceptance Criteria

- [ ] The new bundle's FPC is compared against the aggregate FPC of all conflicting bundles.
- [ ] The comparison is strictly greater than (equal FPC is rejected).
- [ ] FPC computation uses `u128` scaled integers, not floating-point.
- [ ] `MempoolError::RbfFpcNotHigher` is returned when the condition fails.
- [ ] Aggregate FPC is computed as `total_fees * FPC_SCALE / total_cost`, not as an average of individual FPCs.
- [ ] Division by zero is handled (conflicting_cost == 0 should not occur but must not panic).

## Implementation Notes

- **Primary file:** `src/mempool/conflict.rs`
- **FPC_SCALE constant:** Defined in SPEC Section 2.5 as `1_000_000_000_000`.
- **Virtual cost vs cost:** The SPEC references `fee_per_cost` in the Chia comparison. dig-mempool uses `fee_per_virtual_cost` (which includes the spend penalty). The comparison should use `virtual_cost` consistently for both the new item and conflicting items.
- **Edge case — zero-fee conflicting bundles:** If all conflicting bundles have zero fee, their aggregate FPC is 0. Any positive-fee replacement passes this check. A zero-fee replacement must still pass since `0 <= 0` fails the strict comparison — this is intentional (zero-fee items should not be replaceable by other zero-fee items).
- **Chia comparison:** Chia uses `fee_per_cost` (float) at [`mempool_manager.py:1119-1126`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L1119). dig-mempool's scaled integer approach eliminates cross-platform non-determinism.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Higher FPC passes | Unit test | New bundle FPC is 2x the conflicting FPC | RBF FPC check passes |
| Equal FPC rejected | Unit test | New bundle has exactly the same FPC as conflicting | `RbfFpcNotHigher` error |
| Lower FPC rejected | Unit test | New bundle FPC is lower than conflicting | `RbfFpcNotHigher` error |
| Aggregate FPC multiple conflicts | Unit test | Two conflicting bundles with different FPCs; new bundle FPC higher than aggregate | RBF FPC check passes |
| Aggregate FPC barely fails | Unit test | New FPC is higher than one conflict but lower than aggregate of both | `RbfFpcNotHigher` error |
| Zero-fee conflicting | Unit test | Conflicting bundle has 0 fee; new bundle has any positive fee | RBF FPC check passes |
| Scaled integer precision | Unit test | FPC values that would produce different results with float vs integer | Correct integer comparison result |
| Zero-fee replacement of zero-fee | Unit test | Both conflicting and new have 0 fee | `RbfFpcNotHigher` (0 is not strictly greater than 0) |
