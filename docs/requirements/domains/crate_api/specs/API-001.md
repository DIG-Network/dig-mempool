## Summary

The `Mempool` struct provides two constructors: `new()` for creating a mempool with sensible defaults, and `with_config()` for customizing all parameters. Both require `NetworkConstants` from `dig-constants` to establish the network context (genesis challenge, cost limits, AGG_SIG domain data).

This mirrors Chia's `Mempool.__init__()` ([`mempool.py:107-113`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L107)) which accepts `MempoolInfo` containing the pool's configuration (max size, fee threshold, etc.) plus `ConsensusConstants`.

The `new()` constructor derives `max_total_cost` from the network constants: `L2_MAX_COST_PER_BLOCK * MEMPOOL_BLOCK_BUFFER` (550B * 15 = 8.25T by default). All other parameters use the defaults defined in [API-003](API-003.md).

## Specification

### Function Signatures

```rust
impl Mempool {
    /// Create a mempool with default configuration for the given network.
    pub fn new(constants: NetworkConstants) -> Self;

    /// Create a mempool with custom configuration.
    pub fn with_config(constants: NetworkConstants, config: MempoolConfig) -> Self;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `constants` | `NetworkConstants` | From `dig-constants`. Contains consensus constants, genesis challenge, cost limits. |
| `config` | `MempoolConfig` | Custom configuration. See [API-003](API-003.md) for all fields and defaults. |

### new() Behavior

1. Construct `MempoolConfig::default()`.
2. Derive `max_total_cost` from `constants.consensus().max_block_cost_clvm * MEMPOOL_BLOCK_BUFFER` if not set.
3. Initialize all internal data structures (empty pools, empty indexes, empty fee tracker).
4. Initialize BLS cache (empty).
5. Return `Mempool`.

### with_config() Behavior

1. Use the provided `config` as-is.
2. Initialize all internal data structures.
3. Return `Mempool`.

### Thread Safety

The resulting `Mempool` MUST be `Send + Sync`. It uses interior mutability via `RwLock` for all mutable state.

## Acceptance Criteria

- [ ] `Mempool::new()` is publicly exported and accepts `NetworkConstants`.
- [ ] `Mempool::with_config()` is publicly exported and accepts `NetworkConstants` and `MempoolConfig`.
- [ ] `new()` produces a mempool with default configuration.
- [ ] `with_config()` respects all provided configuration values.
- [ ] The returned `Mempool` is `Send + Sync`.
- [ ] Internal state is initialized (empty pools, zero counts).
- [ ] `len()` returns 0 on a newly constructed mempool.
- [ ] `is_empty()` returns true on a newly constructed mempool.

## Implementation Notes

- **Primary file:** `src/mempool.rs`
- **NetworkConstants:** From `dig-constants`. Provides `consensus()` method returning `chia_consensus::ConsensusConstants`.
- **Interior mutability:** The `Mempool` struct uses `&self` methods with internal `RwLock`s (SPEC Section 1.3, Decision #1).
- **Edge cases:**
  - `MempoolConfig` with zero `max_total_cost`: technically valid but makes the pool immediately full. Consider a debug assertion.
