## Summary

`MempoolStats` provides a comprehensive snapshot of current mempool state, returned by `Mempool::stats()`. It aggregates key metrics across the active pool, pending pool, and conflict cache into a single struct for monitoring, logging, and operational decisions.

The stats include counts, cost/fee totals, utilization metrics, fee-per-cost bounds, CPFP dependency statistics, and spend counts. The method targets sub-microsecond latency by reading pre-computed accumulators rather than scanning all items.

---

## Specification

### Struct Definition

```rust
pub struct MempoolStats {
    /// Number of items in the active pool.
    pub active_count: usize,
    /// Number of items in the pending pool.
    pub pending_count: usize,
    /// Number of items in the conflict cache.
    pub conflict_count: usize,
    /// Sum of all active item costs.
    pub total_cost: u64,
    /// Sum of all active item fees.
    pub total_fees: u64,
    /// Maximum total cost (from MempoolConfig).
    pub max_cost: u64,
    /// Current utilization: total_cost / max_cost.
    pub utilization: f64,
    /// Minimum fee_per_virtual_cost_scaled among active items.
    /// 0 if pool is empty.
    pub min_fpc_scaled: u128,
    /// Maximum fee_per_virtual_cost_scaled among active items.
    /// 0 if pool is empty.
    pub max_fpc_scaled: u128,
    /// Number of active items that have dependencies (depth > 0).
    pub items_with_dependencies: usize,
    /// Maximum depth among all active items.
    pub max_current_depth: u32,
    /// Total number of coin spends across all active items.
    pub total_spend_count: usize,
}
```

### Method Signature

```rust
impl Mempool {
    pub fn stats(&self) -> MempoolStats;
}
```

### Performance

`stats()` MUST return in under 1 microsecond (SPEC Section 17). This is achieved by maintaining pre-computed accumulators that are updated on each insertion/removal, rather than scanning all items.

### Field Sources

| Field | Source |
|-------|--------|
| `active_count` | `items.len()` |
| `pending_count` | `pending.len()` |
| `conflict_count` | `conflict_cache.len()` |
| `total_cost` | Running accumulator updated on insert/remove |
| `total_fees` | Running accumulator updated on insert/remove |
| `max_cost` | `config.max_total_cost` |
| `utilization` | `total_cost as f64 / max_cost as f64` |
| `min_fpc_scaled` | Running min tracker |
| `max_fpc_scaled` | Running max tracker |
| `items_with_dependencies` | Count of items where `depth > 0` |
| `max_current_depth` | Running max tracker |
| `total_spend_count` | Running accumulator of `num_spends` |

## Implementation Notes

- **Primary file:** `src/types.rs` and `src/mempool.rs`
- **Accumulators:** Maintain `total_cost`, `total_fees`, `total_spend_count`, `items_with_dependencies`, `max_current_depth` as running values updated on insert/remove.
- **min/max FPC tracking:** Can use a `BTreeMap<u128, usize>` (FPC -> count) to efficiently track min/max without scanning all items. On removal, decrement count; when count reaches 0, remove entry.
- **Locking:** `stats()` acquires read locks on active, pending, and conflict pools.
- **f64 for utilization:** Acceptable for display/monitoring purposes. Not used in any deterministic mempool decision.
- **Edge cases:**
  - Empty pool: utilization = 0.0, min_fpc = 0, max_fpc = 0.
  - Single item: min_fpc == max_fpc.
