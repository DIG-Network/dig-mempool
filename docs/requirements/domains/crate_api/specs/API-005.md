## Summary

`SubmitResult` is the success return type of `submit()` and `submit_with_policy()`. It distinguishes between two successful outcomes: the bundle was admitted to the active mempool (`Success`), or the bundle was valid but timelocked and stored in the pending pool (`Pending`).

On Chia L1, timelocked bundles are routed to the pending cache ([`mempool_manager.py:600-603`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool_manager.py#L600)). The pending result includes the `assert_height` that must be reached before the bundle becomes eligible for promotion to the active pool.

This is a success type -- validation errors, conflicts, and capacity failures are returned as `Err(MempoolError)`.

---

## Specification

### Enum Definition

```rust
pub enum SubmitResult {
    /// Bundle was admitted to the active mempool.
    Success,
    /// Bundle is valid but timelocked; stored in the pending pool.
    Pending { assert_height: u64 },
}
```

### Variant Semantics

| Variant | Meaning | Pool |
|---------|---------|------|
| `Success` | Bundle is in the active pool, eligible for block selection | Active |
| `Pending { assert_height }` | Bundle is valid but not yet eligible; will be promoted when `height >= assert_height` | Pending |

### Return Context

`SubmitResult` is returned inside `Result<SubmitResult, MempoolError>`:
- `Ok(SubmitResult::Success)` -- admitted to active pool.
- `Ok(SubmitResult::Pending { assert_height })` -- admitted to pending pool.
- `Err(MempoolError::*)` -- rejected.

## Acceptance Criteria

- [ ] `SubmitResult` is publicly exported from the crate.
- [ ] Contains `Success` variant with no data.
- [ ] Contains `Pending` variant with `assert_height: u64`.
- [ ] Returned by `submit()`, `submit_with_policy()`, and individual results of `submit_batch()`.
- [ ] `Success` means the item is in the active pool.
- [ ] `Pending` means the item is in the pending pool.

## Implementation Notes

- **Primary file:** `src/types.rs` or `src/submit.rs`
- **Derives:** Consider deriving `Debug`, `Clone`, `PartialEq` for testability.
- **assert_height:** This is the resolved absolute height (after timelock resolution). It is the minimum height at which `on_new_block()` will include this item in `pending_promotions`.
- **Edge cases:**
  - Bundle with both `assert_height` (pending) and `assert_before_height` (expiry): valid combination. The item is stored in pending and may expire before promotion.
