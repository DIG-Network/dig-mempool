## Summary

`MempoolError` is the error type returned by all fallible mempool operations (`submit()`, `submit_batch()`, etc.). It is a comprehensive enum covering all possible failure modes: deduplication, structural checks, CLVM/signature failures, cost/fee violations, conflict/RBF failures, capacity limits, timelock issues, conservation violations, dependency chain errors, spend count limits, policy rejections, and upstream validation errors.

The enum derives `Clone + PartialEq` for testability (SPEC Section 1.3, Design Decision #14). Since `dig-clvm::ValidationError` does not derive `Clone` (it wraps non-Clone types), the `ValidationError` variant stores the error as a `String` via `to_string()`.

Chia uses `chia.util.errors.Err` enum codes (e.g., `DOUBLE_SPEND = 5`, `UNKNOWN_UNSPENT = 6`). dig-mempool uses descriptive Rust enum variants with structured data for better ergonomics.

## Specification

### Enum Definition

```rust
#[derive(Debug, Clone, PartialEq, thiserror::Error)]
pub enum MempoolError {
    // -- Deduplication --
    #[error("bundle already seen: {0}")]
    AlreadySeen(Bytes32),

    // -- Structural --
    #[error("duplicate spend of coin {0} within bundle")]
    DuplicateSpend(Bytes32),

    #[error("coin not found: {0}")]
    CoinNotFound(Bytes32),

    #[error("coin already spent: {0}")]
    CoinAlreadySpent(Bytes32),

    // -- CLVM / signature --
    #[error("CLVM execution error: {0}")]
    ClvmError(String),

    #[error("invalid aggregate signature")]
    InvalidSignature,

    // -- Cost / fee --
    #[error("cost {cost} exceeds maximum {max}")]
    CostExceeded { cost: u64, max: u64 },

    #[error("negative fee: outputs ({output}) exceed inputs ({input})")]
    NegativeFee { input: u64, output: u64 },

    #[error("insufficient fee: required {required}, available {available}")]
    InsufficientFee { required: u64, available: u64 },

    #[error("fee too low for current mempool utilization")]
    FeeTooLow,

    // -- Conflict / RBF --
    #[error("conflicts with existing mempool item {0}")]
    Conflict(Bytes32),

    #[error("RBF rejected: must spend superset of conflicting bundle's coins")]
    RbfNotSuperset,

    #[error("RBF rejected: fee-per-cost not higher than existing")]
    RbfFpcNotHigher,

    #[error("RBF rejected: fee bump {provided} below minimum {required}")]
    RbfBumpTooLow { required: u64, provided: u64 },

    // -- Capacity --
    #[error("mempool full: cannot admit bundle")]
    MempoolFull,

    #[error("pending pool full")]
    PendingPoolFull,

    // -- Timelocks --
    #[error("impossible timelock constraints")]
    ImpossibleTimelocks,

    #[error("bundle has expired")]
    Expired,

    // -- Conservation --
    #[error("conservation violation: input {input}, output {output}")]
    ConservationViolation { input: u64, output: u64 },

    // -- Dependency chains --
    #[error("dependency depth {depth} exceeds maximum {max}")]
    DependencyTooDeep { depth: u32, max: u32 },

    #[error("dependency cycle detected")]
    DependencyCycle,

    // -- Spend count --
    #[error("spend count {count} would exceed block limit {max}")]
    TooManySpends { count: usize, max: usize },

    // -- Policy --
    #[error("admission policy rejected: {0}")]
    PolicyRejected(String),

    // -- Upstream --
    #[error("validation error: {0}")]
    ValidationError(String),
}
```

### Variant Count

24 variants total covering all admission pipeline failure modes.

### Derive Requirements

- `Debug`: Standard error formatting.
- `Clone`: Required for PartialEq and for returning errors from batch operations.
- `PartialEq`: Required for test assertions (`assert_eq!`).
- `thiserror::Error`: Provides `Display` and `Error` trait implementations.

### ValidationError Conversion

```rust
impl From<dig_clvm::ValidationError> for MempoolError {
    fn from(e: dig_clvm::ValidationError) -> Self {
        MempoolError::ValidationError(e.to_string())
    }
}
```

## Implementation Notes

- **Primary file:** `src/error.rs`
- **thiserror:** Add `thiserror` to `Cargo.toml` dependencies.
- **Bytes32 in Clone:** `Bytes32` from `chia-protocol` implements `Clone`, so `AlreadySeen(Bytes32)` is clonable.
- **Error matching:** Consumers can pattern-match on specific variants for error handling (e.g., retry on `MempoolFull`, log on `AlreadySeen`).
