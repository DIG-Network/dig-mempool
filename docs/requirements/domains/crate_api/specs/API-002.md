## Summary

`MempoolItem` is the central data type representing a validated, admitted transaction in the mempool. Items are immutable once created and stored internally as `Arc<MempoolItem>` for cheap sharing across indexes and API consumers.

This corresponds to Chia's `MempoolItem` ([`mempool_item.py:45-120`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L45)) and `BundleCoinSpend` ([`mempool_item.py:25-42`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L25)), flattened into a single struct with CPFP extensions, descendant scoring, and singleton fast-forward support.

Key differences from Chia: integer-scaled FPC (no floats), CPFP package fields, descendant_score for eviction ordering, dependency tracking, and singleton lineage info.

## Specification

### Struct Definition

```rust
pub struct MempoolItem {
    // -- Identity --
    pub spend_bundle: SpendBundle,
    pub spend_bundle_id: Bytes32,

    // -- Cost & fee (individual) --
    pub fee: u64,
    pub cost: u64,
    pub virtual_cost: u64,
    pub fee_per_virtual_cost_scaled: u128,

    // -- Cost & fee (package / CPFP) --
    pub package_fee: u64,
    pub package_virtual_cost: u64,
    pub package_fee_per_virtual_cost_scaled: u128,

    // -- Descendant score (eviction) --
    pub descendant_score: u128,

    // -- State deltas --
    pub additions: Vec<Coin>,
    pub removals: Vec<Bytes32>,

    // -- Metadata --
    pub height_added: u64,
    pub conditions: OwnedSpendBundleConditions,
    pub num_spends: usize,

    // -- Timelocks --
    pub assert_height: Option<u64>,
    pub assert_before_height: Option<u64>,
    pub assert_before_seconds: Option<u64>,

    // -- Dependencies (CPFP) --
    pub depends_on: HashSet<Bytes32>,
    pub depth: u32,

    // -- Deduplication --
    pub eligible_for_dedup: bool,

    // -- Singleton fast-forward --
    pub singleton_lineage: Option<SingletonLineageInfo>,
}
```

### Field Sources

| Field | Source | Notes |
|-------|--------|-------|
| `spend_bundle` | Caller submission | The original `chia-protocol::SpendBundle` |
| `spend_bundle_id` | `SpendBundle::name()` | Chia's canonical bundle hash |
| `fee` | `SpendResult.fee` from dig-clvm | `removal_amount - addition_amount` |
| `cost` | `SpendResult.conditions.cost` | CLVM execution + condition costs |
| `virtual_cost` | Computed | `cost + (num_spends * spend_penalty_cost)` |
| `fee_per_virtual_cost_scaled` | Computed | `fee * FPC_SCALE / virtual_cost` |
| `package_fee` | Computed (CPFP) | `fee + sum(ancestor.fee)` |
| `package_virtual_cost` | Computed (CPFP) | `virtual_cost + sum(ancestor.virtual_cost)` |
| `package_fee_per_virtual_cost_scaled` | Computed (CPFP) | `package_fee * FPC_SCALE / package_virtual_cost` |
| `descendant_score` | Computed | `max(own_fpc, max(descendant.package_fpc))` |
| `additions` | `SpendResult.additions` | Coins created by CREATE_COIN |
| `removals` | `SpendResult.removals` | Coin IDs consumed |
| `height_added` | Caller-provided `current_height` | L2 block height at admission |
| `conditions` | `SpendResult.conditions` | Full CLVM output |
| `num_spends` | `spend_bundle.coin_spends.len()` | Spend count |
| `assert_height` | Timelock resolution | Earliest valid height |
| `assert_before_height` | Timelock resolution | Latest valid height (expiry) |
| `assert_before_seconds` | Timelock resolution | Latest valid timestamp (expiry) |
| `depends_on` | CPFP resolution | Parent bundle IDs |
| `depth` | CPFP resolution | 0 = no dependencies |
| `eligible_for_dedup` | `conditions.spends[*].flags & ELIGIBLE_FOR_DEDUP` | All spends must have flag |
| `singleton_lineage` | Caller-provided or detected | Optional singleton state |

### Storage

Items MUST be stored as `Arc<MempoolItem>` internally. Public API methods return `Arc<MempoolItem>` for zero-copy sharing. Items are immutable once constructed (Chia's `MempoolItem` is a frozen dataclass).

## Implementation Notes

- **Primary file:** `src/types.rs` or `src/item.rs`
- **Arc wrapping:** Construction creates `Arc::new(MempoolItem { ... })`. The `Arc` is what gets stored in all indexes.
- **Descendant score mutability:** Despite items being "immutable", `descendant_score` is updated when children are added/removed. This can be handled via `AtomicU128` or by replacing the `Arc` (since re-insertion is cheap).
- **Type imports:** `SpendBundle`, `Coin`, `Bytes32` from `chia-protocol` (via dig-clvm re-exports). `OwnedSpendBundleConditions` from `chia-consensus`. `HashSet` from `std::collections`.
- **Serialization:** `MempoolItem` needs serde support for [LCY-007](../../lifecycle/specs/LCY-007.md) snapshot/restore.
