## Summary

The `Mempool` struct exposes a comprehensive set of query methods for inspecting mempool state. All query methods are read-only and acquire read locks, enabling concurrent access alongside other reads and Phase 1 validation.

These methods provide the caller with the ability to look up items, enumerate pools, traverse dependency graphs, and inspect coin creation/spending relationships. The CPFP coin queries (`get_mempool_coin_record`, `get_mempool_coin_creator`) are particularly important for building child-pays-for-parent transaction chains.

## Specification

### Item Queries

```rust
impl Mempool {
    /// Look up an active item by bundle ID.
    pub fn get(&self, bundle_id: &Bytes32) -> Option<Arc<MempoolItem>>;

    /// Check if a bundle ID exists in the active pool.
    pub fn contains(&self, bundle_id: &Bytes32) -> bool;
}
```

### Enumeration

```rust
impl Mempool {
    /// Return all active bundle IDs.
    pub fn active_bundle_ids(&self) -> Vec<Bytes32>;

    /// Return all pending bundle IDs.
    pub fn pending_bundle_ids(&self) -> Vec<Bytes32>;

    /// Return all active items.
    pub fn active_items(&self) -> Vec<Arc<MempoolItem>>;
}
```

### Dependency Graph

```rust
impl Mempool {
    /// Return direct dependents (children) of a bundle.
    pub fn dependents_of(&self, bundle_id: &Bytes32) -> Vec<Arc<MempoolItem>>;

    /// Return all ancestors (parents, grandparents, ...) of a bundle.
    pub fn ancestors_of(&self, bundle_id: &Bytes32) -> Vec<Arc<MempoolItem>>;
}
```

### Counts

```rust
impl Mempool {
    /// Number of active items.
    pub fn len(&self) -> usize;

    /// Number of pending items.
    pub fn pending_len(&self) -> usize;

    /// Number of conflict cache items.
    pub fn conflict_len(&self) -> usize;
```

### CPFP Coin Queries

```rust
impl Mempool {
    /// Look up a coin created by an active mempool item and return a
    /// synthetic CoinRecord for use in a subsequent submit() call.
    pub fn get_mempool_coin_record(&self, coin_id: &Bytes32) -> Option<CoinRecord>;

    /// Look up which active mempool item created a given coin.
    pub fn get_mempool_coin_creator(&self, coin_id: &Bytes32) -> Option<Bytes32>;
}
```

**`get_mempool_coin_record()` semantics:**
- Returns a synthetic `CoinRecord` for a coin created by an active mempool item.
- Synthetic record uses: `confirmed_block_index = parent.height_added`, `timestamp = admission_timestamp`, `spent = false`, `coinbase = false`.
- Returns `None` if the coin was not created by any active mempool item.
- Used by callers to build CPFP child transactions.
- Note: TOCTOU safe -- if the parent is evicted between this call and `submit()`, Phase 2 will reject with `CoinNotFound`.

**`get_mempool_coin_creator()` semantics:**
- Returns the bundle ID of the active item that created the given coin.
- Returns `None` if no active item created it.

- [ ] `ancestors_of()` returns full ancestor chain.

## Implementation Notes

- **Primary file:** `src/mempool.rs` (query impl block)
- **Locking:** All methods acquire read locks. `get()` and `contains()` target < 1us (SPEC Section 17). `active_items()` and `active_bundle_ids()` may be slower for large pools (clone all Arc pointers).
- **Dependency traversal:** `ancestors_of()` walks the `dependencies` graph upward. `dependents_of()` reads the `dependents` map. Both are bounded by `max_dependency_depth`.
- **CPFP coin lookup:** Uses the `mempool_coins` index (`coin_id -> creating_bundle_id`).
- **Edge cases:**
  - `get()` for non-existent ID: returns `None`.
  - `dependents_of()` for item with no children: returns empty `Vec`.
  - `ancestors_of()` for root item (depth 0): returns empty `Vec`.
  - `get_mempool_coin_record()` for on-chain coin: returns `None` (only mempool-created coins).
