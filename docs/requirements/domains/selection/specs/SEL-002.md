# SEL-002: Pre-Selection Filtering

> **Authoritative requirement:** [SEL-002](../NORMATIVE.md#SEL-002)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 6.1

---

## Summary

Before any selection strategy runs, the candidate set must be filtered to exclude items that are no longer valid at the current height and timestamp. This includes items that have expired (their `assert_before_height` or `assert_before_seconds` has been reached) and items that are still future-timelocked (their `assert_height` has not yet been reached).

This filtering corresponds to the implicit filtering in Chia's SQL-based selection, where expired items would already have been removed by `new_tx_block()` ([`mempool.py:336-340`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L336)). dig-mempool performs this filtering explicitly at selection time as a defense-in-depth measure, ensuring no stale items leak into block candidates even if `on_new_block()` was called with a slightly older height/timestamp.

---

## Specification

### Filtering Rules

An item is **excluded** from the candidate set if any of the following conditions hold:

| Condition | Meaning | Filter |
|-----------|---------|--------|
| `item.assert_before_height.is_some() && item.assert_before_height.unwrap() <= height` | Item has expired by height | Exclude |
| `item.assert_before_seconds.is_some() && item.assert_before_seconds.unwrap() <= timestamp` | Item has expired by time | Exclude |
| `item.assert_height.is_some() && item.assert_height.unwrap() > height` | Item is future-timelocked | Exclude |

### Semantics

- `assert_before_height` is a **strict upper bound**: the item is valid only when `height < assert_before_height`. When `height >= assert_before_height`, the item has expired.
- `assert_before_seconds` is a **strict upper bound**: the item is valid only when `timestamp < assert_before_seconds`. When `timestamp >= assert_before_seconds`, the item has expired.
- `assert_height` is a **minimum**: the item is valid only when `height >= assert_height`. When `height < assert_height`, the item is not yet valid.
- Items with `None` for any of these fields are unconstrained along that dimension.

### Output

The filtered candidate set is passed to all four selection strategies ([SEL-003](SEL-003.md) through [SEL-006](SEL-006.md)). Each strategy operates on the same filtered set.

---

## Acceptance Criteria

- [ ] Items with `assert_before_height <= height` are excluded from selection.
- [ ] Items with `assert_before_seconds <= timestamp` are excluded from selection.
- [ ] Items with `assert_height > height` (future-timelocked) are excluded from selection.
- [ ] Items with `None` timelock fields are not filtered out.
- [ ] Filtering occurs before any strategy sort, not during greedy accumulation.
- [ ] The same filtered set is used by all four strategies.

---

## Implementation Notes

- **Primary file:** `src/selection/filter.rs` or inline in `src/selection/mod.rs`
- **Performance:** The filter is a linear scan over active items. With a typical mempool of 10K-50K items, this completes in microseconds.
- **Defense-in-depth:** Even though `on_new_block()` removes expired items, the filter catches any items that expired between the last `on_new_block()` call and the current `select_for_block()` call.
- **Edge case:** An item could have both `assert_height` and `assert_before_height`. If `assert_height > height`, it is excluded (future-timelocked). If `assert_before_height <= height`, it is excluded (expired). Both checks apply independently.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Expired by height excluded | Unit test | Item with `assert_before_height = 100`, call with `height = 100` | Item excluded |
| Expired by time excluded | Unit test | Item with `assert_before_seconds = 5000`, call with `timestamp = 5000` | Item excluded |
| Future-timelocked excluded | Unit test | Item with `assert_height = 200`, call with `height = 150` | Item excluded |
| Valid item included | Unit test | Item with `assert_height = 100`, `assert_before_height = 200`, call with `height = 150` | Item included |
| None timelocks not filtered | Unit test | Item with all timelock fields `None` | Item included |
| Both expired and timelocked | Unit test | Item with `assert_height = 200` and `assert_before_height = 100`, call with `height = 150` | Item excluded (expired) |
| Boundary: assert_before exactly at height | Unit test | `assert_before_height = 100`, `height = 100` | Excluded (100 <= 100) |
| Boundary: assert_height exactly at height | Unit test | `assert_height = 100`, `height = 100` | Included (100 >= 100) |
