## Summary

After all four selection strategies ([SEL-003](SEL-003.md) through [SEL-006](SEL-006.md)) produce their candidate sets, the best-selection comparator chooses the single best set to use for block production. The comparison uses a lexicographic ordering: highest total fees first, then lowest total virtual cost, then fewest bundles.

This multi-strategy approach is a dig-mempool extension over Chia L1, which uses a single strategy (`ORDER BY priority DESC, seq ASC` at [`mempool.py:605`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L605)). By running four strategies and picking the best result, dig-mempool adapts to different mempool compositions without requiring the block producer to manually tune selection parameters.

## Specification

### Comparison Algorithm

Given four candidate sets `S1, S2, S3, S4` (from strategies 1-4 respectively), the comparator selects the best set using the following lexicographic ordering:

1. **Highest total fees:** `total_fees(Si) = sum(item.fee for item in Si)`. The set with the highest total fees wins.
2. **Lowest total virtual cost** (tiebreaker): If two sets have equal total fees, the set with the lower total virtual cost wins (it leaves more room in the block for future use or is more efficient).
3. **Fewest bundles** (tiebreaker): If two sets have equal total fees and total virtual cost, the set with fewer bundles wins (simpler block, less overhead).

### Formal Definition

```
best(S1, S2, S3, S4) = argmax over Si of:
    (total_fees(Si), -total_virtual_cost(Si), -bundle_count(Si))
```

Where comparison is lexicographic on the tuple.

### Edge Cases

- If all four strategies produce empty sets (no eligible items), the comparator returns an empty set.
- If multiple strategies produce identical candidate sets (same items in different order), they will have identical comparison values. The comparator returns S1 (Strategy 1) as the default winner in ties.
- Total fees are computed as `u64` sums. Overflow is not a concern because `max_total_cost * max_fee_per_cost` fits in `u64` for realistic parameters.

---

## Acceptance Criteria

- [ ] All four strategy candidate sets are computed before comparison.
- [ ] The set with the highest total fees is selected.
- [ ] If total fees are equal, the set with the lowest total virtual cost is selected.
- [ ] If total fees and total virtual cost are equal, the set with the fewest bundles is selected.
- [ ] If all comparison values are equal, Strategy 1's set is returned as the default.
- [ ] The comparator is deterministic.

---

## Implementation Notes

- **Primary file:** `src/selection/best.rs` or `src/selection/mod.rs`
- **Performance:** Each strategy produces at most ~6000 items (max_spends_per_block). Computing total_fees and total_virtual_cost is O(n) per strategy, so the comparison is O(4n) which is negligible.
- **Metrics:** The implementation should log which strategy won (for operator visibility and tuning).
- **Custom strategy:** `select_for_block_with_strategy()` bypasses the 4-way comparison entirely and delegates to a caller-provided `BlockSelectionStrategy`. The comparator is not used in that path.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Highest fees wins | Unit test | S1 (fees=100), S2 (fees=200), S3 (fees=150), S4 (fees=180) | S2 selected |
| Fee tie, lowest cost wins | Unit test | S1 (fees=100, cost=500), S2 (fees=100, cost=300) | S2 selected |
| Fee+cost tie, fewest bundles wins | Unit test | S1 (fees=100, cost=300, count=5), S2 (fees=100, cost=300, count=3) | S2 selected |
| All equal, Strategy 1 default | Unit test | All four identical sets | S1 returned |
| All empty sets | Unit test | No eligible items | Empty set returned |
| Single non-empty strategy | Unit test | Only S3 produces items, others empty | S3 selected |
| Whale beats density on fees | Integration test | Whale strategy (S2) yields 10M fees, density (S1) yields 8M | S2 selected |
| Age-weighted competitive | Integration test | Old moderate-fee items yield more total fee than new high-fee items | S4 selected |
