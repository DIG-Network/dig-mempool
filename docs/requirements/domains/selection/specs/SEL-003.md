# SEL-003: Strategy 1 -- Fee-Per-Cost Density Sort

## Summary

Strategy 1 maximizes the total fees collected per unit of block cost consumed. It sorts candidates by fee-per-virtual-cost density, using CPFP package rates for items with dependencies. This is the closest analog to Chia's single `ORDER BY priority DESC, seq ASC` selection ([`mempool.py:605`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L605)), where `priority` maps to fee-per-cost and `seq` maps to insertion order.

The strategy uses CPFP-aware greedy accumulation: when a child item is selected, all its unselected ancestors are automatically included. If the combined cost of the child plus its unselected ancestors would exceed the remaining budget or conflict with already-selected items, the child is skipped.

## Specification

### Sort Order

Candidates are sorted by the following keys, in order:

| Priority | Field | Direction | Rationale |
|----------|-------|-----------|-----------|
| 1 | `package_fee_per_virtual_cost_scaled` | Descending | Highest density first |
| 2 | `fee` | Descending | Higher absolute fee wins ties |
| 3 | `virtual_cost` | Ascending | Smaller items win ties (more room for others) |
| 4 | `height_added` | Ascending | Older items win ties (fairness) |
| 5 | `spend_bundle_id` | Ascending | Deterministic tiebreaker |

### CPFP-Aware Greedy Accumulation

For each item in sorted order:

1. Compute the set of **unselected ancestors** (items in `depends_on` chain not yet selected).
2. Compute `total_cost = item.virtual_cost + sum(ancestor.virtual_cost for unselected ancestors)`.
3. Compute `total_spends = item.num_spends + sum(ancestor.num_spends for unselected ancestors)`.
4. **Skip** if `total_cost > remaining_block_cost`.
5. **Skip** if adding `total_spends` would exceed `config.max_spends_per_block`.
6. **Skip** if any unselected ancestor or the item itself conflicts with an already-selected item (shared spent coin).
7. Otherwise, **include** the item and all its unselected ancestors.

### Package Fee Rate

For items with CPFP dependencies (`depends_on` is non-empty), `package_fee_per_virtual_cost_scaled` aggregates the fees and costs of the entire ancestor chain:

```
package_fee = fee + sum(ancestor.fee)
package_virtual_cost = virtual_cost + sum(ancestor.virtual_cost)
package_fee_per_virtual_cost_scaled = (package_fee * FPC_SCALE) / package_virtual_cost
```

For items without dependencies, `package_fee_per_virtual_cost_scaled == fee_per_virtual_cost_scaled`.

## Acceptance Criteria

- [ ] Candidates are sorted by package FPC descending as the primary key.
- [ ] Secondary keys are fee desc, virtual_cost asc, height_added asc, bundle_id asc.
- [ ] Greedy accumulation automatically includes unselected ancestors with child.
- [ ] Items exceeding remaining cost budget are skipped.
- [ ] Items exceeding spend count limit are skipped.
- [ ] Conflicting items are skipped.
- [ ] The sort is deterministic (same input always produces same output).

---

## Implementation Notes

- **Primary file:** `src/selection/strategies.rs`
- **Package FPC:** Items with dependencies use `package_fee_per_virtual_cost_scaled` (pre-computed at admission time, stored on `MempoolItem`).
- **Ancestor tracking:** Maintain a `HashSet<Bytes32>` of selected bundle IDs. For each candidate, walk `depends_on` to find unselected ancestors.
- **Conflict detection:** Maintain a `HashSet<Bytes32>` of spent coin IDs across all selected items. Check each candidate's and ancestor's `removals` against this set.
