# SEL-004: Strategy 2 -- Absolute Fee Whale Sort

## Summary

Strategy 2 prioritizes transactions with the highest absolute fees, regardless of their cost density. This benefits "whale" transactions -- large, high-fee bundles that pay substantial total fees but may have lower fee-per-cost ratios than smaller transactions.

Chia's single-strategy selection (`ORDER BY priority DESC` at [`mempool.py:605`](https://github.com/Chia-Network/chia-blockchain/blob/main/chia/full_node/mempool.py#L605)) inherently favors density over absolute fee. dig-mempool includes this whale strategy to ensure high-value transactions are not starved by many small high-density transactions. The best-selection comparator ([SEL-007](SEL-007.md)) chooses whichever strategy yields the highest total fees.

## Specification

### Sort Order

Candidates are sorted by the following keys, in order:

| Priority | Field | Direction | Rationale |
|----------|-------|-----------|-----------|
| 1 | `package_fee` | Descending | Highest absolute fee first |
| 2 | `package_fee_per_virtual_cost_scaled` | Descending | Among equal-fee items, prefer higher density |
| 3 | `virtual_cost` | Ascending | Smaller items win ties |
| 4 | `height_added` | Ascending | Older items win ties (fairness) |
| 5 | `spend_bundle_id` | Ascending | Deterministic tiebreaker |

### CPFP-Aware Greedy Accumulation

Identical to [SEL-003](SEL-003.md) greedy accumulation:

1. Compute unselected ancestors.
2. Compute total cost = item + unselected ancestors.
3. Compute total spends = item + unselected ancestors.
4. Skip if total cost > remaining block cost.
5. Skip if total spends would exceed `config.max_spends_per_block`.
6. Skip if any conflict with already-selected items.
7. Otherwise include item + all unselected ancestors.

### Package Fee

For items with CPFP dependencies, `package_fee` is the sum of fees across the item and all its ancestors:

```
package_fee = fee + sum(ancestor.fee)
```

For items without dependencies, `package_fee == fee`.

## Acceptance Criteria

- [ ] Candidates are sorted by package fee descending as the primary key.
- [ ] Secondary keys are package FPC desc, virtual_cost asc, height_added asc, bundle_id asc.
- [ ] Greedy accumulation automatically includes unselected ancestors with child.
- [ ] Items exceeding remaining cost budget are skipped.
- [ ] Items exceeding spend count limit are skipped.
- [ ] Conflicting items are skipped.
- [ ] The sort is deterministic.

## Implementation Notes

- **Primary file:** `src/selection/strategies.rs`
- **When whale wins:** This strategy outperforms density sort when a single large transaction pays more total fee than many small high-FPC transactions combined. Example: one 100M-cost, 10M-fee tx vs. twenty 5M-cost, 400K-fee txs. The whale tx collects 10M vs. the twenty collecting 8M.
- **Package fee pre-computed:** `package_fee` is stored on `MempoolItem` at admission time.

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Higher fee selected first | Unit test | A (fee=10M, FPC=50), B (fee=500K, FPC=100). Budget fits both. | A selected before B |
| FPC tiebreaker | Unit test | Same fee, A (FPC=100), B (FPC=50) | A selected before B |
| Whale vs many small | Unit test | 1 whale (fee=10M, cost=200M) vs 20 small (fee=400K each, cost=5M each). Budget=200M | Whale selected (10M > 8M total from small) |
| CPFP package fee | Unit test | Parent P (fee=1M), child C (fee=5M). C's package_fee=6M. | C (with P) sorted by package_fee=6M |
| Budget constraint | Unit test | Whale exceeds budget | Whale skipped, smaller items fill budget |
| Conflict skip | Unit test | Whale conflicts with already-selected item | Whale skipped |
| Deterministic ordering | Unit test | Run twice with identical input | Identical output |
| Spend count limit | Unit test | Whale has many spends exceeding limit | Whale skipped |
