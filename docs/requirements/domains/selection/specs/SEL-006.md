# SEL-006: Strategy 4 -- Age-Weighted Sort

> **Authoritative requirement:** [SEL-006](../NORMATIVE.md#SEL-006)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Strategy 4 prevents transaction starvation by prioritizing the oldest items in the mempool. Items that have been waiting the longest are selected first, with fee-per-cost as a secondary tiebreaker. This ensures that low-fee transactions eventually get included even when the mempool is congested with higher-fee transactions.

This is a dig-mempool extension not present in Chia L1. Chia's single `ORDER BY priority DESC` strategy ([`mempool.py:605`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L605)) can starve low-priority transactions indefinitely. The age-weighted strategy addresses this by giving older transactions a chance to be included.

The best-selection comparator ([SEL-007](SEL-007.md)) will only choose this strategy's output if it yields higher total fees than the other three strategies -- so age-weighting only wins when it happens to produce a more profitable block (e.g., when old items have accumulated enough fee to be competitive).

## Specification

### Sort Order

Candidates are sorted by the following keys, in order:

| Priority | Field | Direction | Rationale |
|----------|-------|-----------|-----------|
| 1 | `height_added` | Ascending | Oldest items first (anti-starvation) |
| 2 | `package_fee_per_virtual_cost_scaled` | Descending | Among same-age items, highest density first |
| 3 | `fee` | Descending | Higher fee wins ties |
| 4 | `spend_bundle_id` | Ascending | Deterministic tiebreaker |

### CPFP-Aware Greedy Accumulation

Identical to [SEL-003](SEL-003.md) greedy accumulation:

1. Compute unselected ancestors.
2. Compute total cost = item + unselected ancestors.
3. Compute total spends = item + unselected ancestors.
4. Skip if total cost > remaining block cost.
5. Skip if total spends would exceed `config.max_spends_per_block`.
6. Skip if any conflict with already-selected items.
7. Otherwise include item + all unselected ancestors.

### Age Semantics

`height_added` is the L2 block height at which the item was admitted to the mempool. Lower values indicate older items. Two items admitted during the same block height are distinguished by their fee rate and bundle ID.

## Acceptance Criteria

- [ ] Candidates are sorted by `height_added` ascending as the primary key (oldest first).
- [ ] Secondary keys are package FPC desc, fee desc, bundle_id asc.
- [ ] Greedy accumulation automatically includes unselected ancestors with child.
- [ ] Items exceeding remaining cost budget are skipped.
- [ ] Items exceeding spend count limit are skipped.
- [ ] Conflicting items are skipped.
- [ ] The sort is deterministic.

## Implementation Notes

- **Primary file:** `src/selection/strategies.rs`
- **When age wins:** This strategy produces higher total fees than density sort when the mempool contains many old moderate-fee transactions that together yield more total fee than a few new high-fee transactions. The comparator in [SEL-007](SEL-007.md) ensures this strategy is only chosen when it actually produces a better block.
- **Starvation prevention:** Even if this strategy's candidate set is not chosen as the best, its existence in the 4-way comparison raises the bar -- any strategy that systematically starves old transactions must also beat the age-weighted alternative on total fees.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Oldest item selected first | Unit test | A (height_added=10), B (height_added=50). Both fit budget. | A selected before B |
| FPC tiebreaker at same age | Unit test | A (height=10, FPC=100), B (height=10, FPC=50) | A selected before B |
| Old low-fee vs new high-fee | Unit test | Old (height=5, fee=1M) and new (height=100, fee=5M). Both fit. | Old selected first |
| CPFP with old parent | Unit test | Old parent (height=5) + new child (height=100) | Parent selected first (oldest), child follows via CPFP |
| Budget constraint | Unit test | Old items fill budget before reaching new items | New items excluded |
| Conflict with old item | Unit test | Old item conflicts with already-selected | Old item skipped despite age |
| Deterministic output | Unit test | Run twice, same input | Identical results |
| Bundle ID tiebreaker | Unit test | Same height, same FPC, same fee, different IDs | Lower bundle_id first |
