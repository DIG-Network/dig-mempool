# CPF-003: Maximum Dependency Depth Enforcement

> **Authoritative requirement:** [CPF-003](../NORMATIVE.md#CPF-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 5.8

---

## Summary

To prevent excessively long dependency chains that would degrade mempool performance (cascade eviction cost, package fee computation cost, block selection complexity), the mempool enforces a maximum dependency depth. After dependency resolution (CPF-002) computes the new item's depth, the mempool checks whether it exceeds the configured limit.

The depth limit is configurable via `MempoolConfig.max_dependency_depth` with a default of 25. A depth of 0 means the item has no dependencies (all coins are on-chain). A depth of 1 means the item depends on one layer of parents. The check is `depth > max_dependency_depth`.

This is a dig-mempool extension. Chia L1 does not support CPFP and therefore has no depth limit. The default value of 25 balances flexibility for complex transaction patterns against the O(depth) cost of operations that traverse ancestor chains.

---

## Specification

### Constants

```rust
pub const DEFAULT_MAX_DEPENDENCY_DEPTH: u32 = 25;
```

### Configuration

```rust
pub struct MempoolConfig {
    /// Maximum dependency chain depth. Default: 25.
    pub max_dependency_depth: u32,
    // ...
}
```

### Algorithm

After dependency resolution (CPF-002) computes `item.depth`:
```rust
if item.depth > config.max_dependency_depth {
    return Err(MempoolError::DependencyTooDeep {
        depth: item.depth,
        max: config.max_dependency_depth,
    });
}
```

### Error Type

```rust
MempoolError::DependencyTooDeep { depth: u32, max: u32 }
```

---

## Acceptance Criteria

- [ ] Items with depth <= `max_dependency_depth` are accepted.
- [ ] Items with depth > `max_dependency_depth` are rejected with `MempoolError::DependencyTooDeep`.
- [ ] The error includes both the actual depth and the maximum.
- [ ] The default `max_dependency_depth` is 25.
- [ ] `max_dependency_depth` is configurable via `MempoolConfig`.
- [ ] Items with depth 0 (no dependencies) always pass this check.

---

## Implementation Notes

- **Primary file:** `src/mempool/cpfp.rs`
- **Depth computation:** Depth is computed during CPF-002. This check follows immediately.
- **Performance rationale:** Operations that traverse ancestor chains (package fee computation, cascade eviction, block selection ancestor accumulation) have O(depth * branching_factor) cost. Bounding depth at 25 keeps these operations fast.
- **Edge case — depth exactly at limit:** `depth == max_dependency_depth` is accepted (the check is strictly greater than).
- **Configuration interaction:** Setting `max_dependency_depth = 0` effectively disables CPFP (no dependent items allowed).

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Depth 0 accepted | Unit test | Item with no dependencies | Accepted (depth 0 <= 25) |
| Depth 1 accepted | Unit test | Single parent-child chain | Accepted (depth 1 <= 25) |
