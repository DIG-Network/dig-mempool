# CPF-001: mempool_coins Index

> **Authoritative requirement:** [CPF-001](../NORMATIVE.md#CPF-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Sections 5.8, 8.1

## Summary

The `mempool_coins` index is the foundational data structure that enables Child-Pays-For-Parent (CPFP) dependency tracking. It maps every coin created by an active mempool item to the bundle that created it. When a new bundle attempts to spend a coin that is not on-chain (not in the caller's `coin_records`), the mempool consults this index to determine whether the coin exists as an unconfirmed output of another mempool item.

This index is a dig-mempool extension not present in Chia L1 (Chia does not support CPFP dependency chains). It is the CPFP analog of `coin_index` (which maps spent coins to spending bundles for conflict detection).

The index must be maintained consistently: entries are added when items are admitted and removed when items are evicted, confirmed, or cascade-evicted. Failure to maintain this invariant would result in false `CoinNotFound` errors for valid CPFP children or, worse, phantom dependencies on evicted parents.

## Specification

### Data Structure

```rust
// coin_id -> creating bundle_id
mempool_coins: HashMap<Bytes32, Bytes32>
```

### Insertion

When a `MempoolItem` is added to the active pool:
```rust
for coin in &item.additions {
    let coin_id = coin.coin_id();
    mempool_coins.insert(coin_id, item.spend_bundle_id);
}
```

### Removal

When a `MempoolItem` is removed from the active pool (for any reason):
```rust
for coin in &item.additions {
    let coin_id = coin.coin_id();
    mempool_coins.remove(&coin_id);
}
```

### Query

```rust
pub fn get_mempool_coin_creator(&self, coin_id: &Bytes32) -> Option<Bytes32> {
    self.mempool_coins.get(coin_id).copied()
}

pub fn get_mempool_coin_record(&self, coin_id: &Bytes32) -> Option<CoinRecord> {
    // Synthetic CoinRecord for CPFP coins
    // confirmed_block_index = parent's height_added
    // timestamp = parent's admission timestamp
    // spent = false
    // coinbase = false
}
```

## Acceptance Criteria

- [ ] Every coin in `item.additions` is registered in `mempool_coins` on active pool insertion.
- [ ] The mapping is `coin_id -> creating bundle_id`.
- [ ] Entries are removed when the creating item is removed, evicted, confirmed, or cascade-evicted.
- [ ] `get_mempool_coin_creator()` returns `Some(bundle_id)` for coins created by active items.
- [ ] `get_mempool_coin_creator()` returns `None` for coins not created by any active item.
- [ ] `get_mempool_coin_record()` returns a synthetic `CoinRecord` with correct fields.
- [ ] No stale entries remain after item removal (verified by post-removal query).

## Implementation Notes

- **Primary file:** `src/mempool/pool.rs` (index maintenance during insert/remove)
- **Public API:** `get_mempool_coin_record()` and `get_mempool_coin_creator()` are public methods on `Mempool` (SPEC Section 3.3).
- **Lock scope:** `mempool_coins` is protected by `pool_lock` (same as `items` and `coin_index`).
- **Coin ID computation:** Uses `Coin::coin_id()` from `chia-protocol`, which computes `sha256(parent_coin_id || puzzle_hash || amount)`.
- **TOCTOU safety:** As noted in SPEC Section 3.3, `get_mempool_coin_record()` is not TOCTOU-safe — if the parent is evicted between this call and the child's `submit()`, Phase 2 rejects the child with `CoinNotFound`. This is acceptable.
- **Overlap with coin_index:** `coin_index` maps spent coins to their spending bundle; `mempool_coins` maps created coins to their creating bundle. Both are populated from the same `MempoolItem` but track different relationships.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Additions registered on insert | Unit test | Insert item with 3 additions; query each coin_id | All 3 return creating bundle_id |
| Entries removed on eviction | Unit test | Insert item, then remove it; query additions | All return None |
| get_mempool_coin_record fields | Unit test | Query a mempool coin; verify CoinRecord fields | correct coin, height_added, spent=false, coinbase=false |
| get_mempool_coin_creator returns None for unknown | Unit test | Query a coin_id not created by any item | Returns None |
| No stale entries after cascade eviction | Unit test | Parent creates coin X, child spends X; evict parent | X removed from mempool_coins |
| Multiple items same additions impossible | Unit test | Two different bundles cannot create the same coin (different parent_ids) | No collision in mempool_coins |
| RBF replacement cleans old entries | Unit test | Item A creates coin X; RBF replaces A with B (different additions) | X removed; B's additions registered |
| Batch insert registers all | Unit test | submit_batch with 3 items; verify all additions indexed | All additions present in mempool_coins |
