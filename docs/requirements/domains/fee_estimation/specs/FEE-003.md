## Summary

`estimate_fee_rate(target_blocks)` provides callers with a fee rate estimate for confirmation within a target number of blocks. It returns an `Option<FeeRate>` where `FeeRate` is the `chia-protocol` type containing `mojos_per_clvm_cost: u64`.

The estimation algorithm scans fee-rate buckets from highest to lowest, computing a success rate (confirmed transactions / total observed) for each bucket at the given target. The first bucket that meets the 85% confidence threshold is returned as the recommended fee rate. This mirrors Chia's `SmartFeeEstimator` which uses the `FeeTracker` data for block-target estimation.

When insufficient data has been collected (fewer than `fee_estimator_window / 2` blocks tracked), the function returns `None` to indicate that no reliable estimate can be made.

---

## Specification

### Function Signature

```rust
impl Mempool {
    pub fn estimate_fee_rate(&self, target_blocks: u32) -> Option<FeeRate>;
}
```

### Algorithm

1. If `block_history.len() < window / 2`, return `None` (insufficient data).
2. For each bucket from highest fee rate to lowest:
   - Select the appropriate confirmation counter based on `target_blocks`:
     - target <= 1: use `confirmed_in_1`
     - target <= 2: use `confirmed_in_2`
     - target <= 5: use `confirmed_in_5`
     - target <= 10: use `confirmed_in_10`
     - target > 10: use `confirmed_in_10` (best available)
   - Compute `success_rate = confirmed_in_N / total_observed`.
   - If `total_observed < 1.0`, skip this bucket (no data).
   - If `success_rate >= 0.85`, this bucket's lower bound fee rate is sufficient.
3. Return the lower bound of the first sufficient bucket as a `FeeRate`.
4. If no bucket meets the threshold, return `None`.

### Return Type

`Option<FeeRate>` where `FeeRate` is `chia_protocol::FeeRate { mojos_per_clvm_cost: u64 }`.

The scaled FPC value from the bucket is converted to `mojos_per_clvm_cost` by dividing by `FPC_SCALE`.

## Acceptance Criteria

- [ ] `estimate_fee_rate()` is publicly exported on the `Mempool` struct.
- [ ] Returns `Option<FeeRate>` using the `chia-protocol` `FeeRate` type.
- [ ] Returns `None` when fewer than `window / 2` blocks have been tracked.
- [ ] Uses 85% confidence threshold for bucket selection.
- [ ] Scans buckets from highest to lowest fee rate.
- [ ] Maps `target_blocks` to the nearest available confirmation counter.
- [ ] Reads fee tracker state under a read lock (concurrent with other operations).

## Implementation Notes

- **Primary file:** `src/fee/estimation.rs`
- **Thread safety:** Reads `fee_lock` (RwLock read). Concurrent with `submit()` and `select_for_block()`.
- **FeeRate conversion:** `mojos_per_clvm_cost = bucket.fee_rate_lower / FPC_SCALE`. Since `FPC_SCALE = 1e12` and fee rates are stored as `u128`, the division may lose precision for very low fee rates.
- **Target mapping:** Targets above 10 use the `confirmed_in_10` counter. Callers requesting very long targets should understand the estimator's resolution limit.
- **Edge cases:**
  - `target_blocks = 0`: treated as `target_blocks = 1`.
  - All buckets empty: returns `None`.
  - Only high-fee-rate buckets have data: returns a high fee rate (conservative).
