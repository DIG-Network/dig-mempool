## Summary

`estimate_min_fee(cost, num_spends)` provides callers with the minimum fee required for a transaction of a given cost to be admitted to the mempool under current utilization conditions. This is the primary fee guidance API for wallet software and transaction builders.

The function implements a 3-tier system based on mempool utilization (total active cost / max total cost). This mirrors Chia's `get_min_fee_rate()` ([`mempool.py:301-327`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool.py#L301)) which returns 0 when not at capacity and iterates ascending `fee_per_cost` to find the eviction threshold when full. dig-mempool simplifies this into explicit utilization tiers with deterministic thresholds.

The function computes `virtual_cost = cost + (num_spends * spend_penalty_cost)` internally, matching Chia's `MempoolItem.virtual_cost` ([`mempool_item.py:92-93`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/types/mempool_item.py#L92)), and uses virtual cost (not raw cost) in the fee calculation.

## Specification

### Function Signature

```rust
impl Mempool {
    pub fn estimate_min_fee(&self, cost: u64, num_spends: usize) -> u64;
}
```

### 3-Tier Fee System

| Utilization | Condition | Minimum Fee |
|-------------|-----------|-------------|
| < 80% | `total_cost < max_total_cost * 0.80` | `0` |
| 80% - 100% | `total_cost >= max_total_cost * 0.80` and `total_cost < max_total_cost` | `virtual_cost * full_mempool_min_fpc_scaled / FPC_SCALE` |
| >= 100% | `total_cost >= max_total_cost` | `virtual_cost * (lowest_fpc + 1) / FPC_SCALE` |

Where:
- `virtual_cost = cost + (num_spends * config.spend_penalty_cost)`
- `full_mempool_min_fpc_scaled` defaults to `5 * FPC_SCALE` (from `MempoolConfig`)
- `lowest_fpc` is the minimum `fee_per_virtual_cost_scaled` among all active items
- `FPC_SCALE = 1_000_000_000_000` (Section 2.5 of SPEC)

### Behavior

- The function reads active pool state under a read lock.
- Utilization is computed as `total_cost / max_total_cost` where `total_cost` is the sum of all active item costs and `max_total_cost` is from `MempoolConfig`.
- At tier 2 (80-100%), the nonzero fee minimum matches Chia's `nonzero_fee_minimum_fpc = 5` ([`mempool_manager.py:746`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/mempool_manager.py#L746)).
- At tier 3 (>=100%), the fee must exceed the lowest item to guarantee admission via eviction.
- Zero-fee transactions are explicitly allowed when utilization is below 80%, matching Chia L1 behavior (Design Decision #12 in SPEC Section 1.3).

## Acceptance Criteria

- [ ] `estimate_min_fee()` is publicly exported on the `Mempool` struct.
- [ ] Returns `0` when mempool utilization is below 80%.
- [ ] Returns `virtual_cost * full_mempool_min_fpc_scaled / FPC_SCALE` when utilization is 80%-100%.
- [ ] Returns `virtual_cost * (lowest_fpc + 1) / FPC_SCALE` when utilization is at or above 100%.
- [ ] Uses `virtual_cost` (not raw cost) in all fee calculations.
- [ ] Reads state under a read lock (concurrent with other reads).
- [ ] Returns `0` when the active pool is empty regardless of tier.

---

## Implementation Notes

- **Primary file:** `src/fee/estimation.rs` (or `src/mempool.rs` fee methods)
- **Dependencies:** Reads `total_cost`, `max_total_cost`, and `lowest_fpc` from pool accumulators.
- **Edge cases:**
  - Empty mempool: utilization is 0%, returns 0.
  - `num_spends = 0`: virtual_cost equals cost (degenerate but valid).
  - Single item in pool at >=100%: `lowest_fpc` is that item's FPC.
  - Integer overflow: `virtual_cost * full_mempool_min_fpc_scaled` uses `u128` arithmetic.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| Empty mempool returns zero | Unit test | Call with empty pool, cost=1000, spends=1 | Returns 0 |
| Below 80% returns zero | Unit test | Fill pool to 79% utilization, query fee | Returns 0 |
| At 80% returns nonzero minimum | Unit test | Fill pool to exactly 80%, query fee | Returns `virtual_cost * 5 * FPC_SCALE / FPC_SCALE` |
| Between 80-100% scales with cost | Unit test | Fill to 90%, query with different costs | Fee proportional to virtual_cost |
| At 100% returns lowest_fpc + 1 | Unit test | Fill pool to 100%, insert items with known FPCs | Returns `virtual_cost * (min_fpc + 1) / FPC_SCALE` |
| Virtual cost includes spend penalty | Unit test | Compare result with num_spends=1 vs num_spends=10 | Higher fee for more spends at same cost |
| Concurrent read safety | Integration test | Call estimate_min_fee while submit is in progress | No deadlock, returns consistent value |
