## Summary

`FeeEstimatorState` is a serializable representation of the `FeeTracker`'s internal state, enabling the fee estimator to survive mempool restarts without losing historical fee data. It is included in `MempoolSnapshot` (SPEC Section 3.9) as the `fee_estimator_state` field.

Without persistence, a restarted mempool would have no fee estimation capability until it has observed `window / 2` blocks (default: 50 blocks). Serialization allows the fee estimator to resume immediately with historical data intact.

The struct derives `serde::Serialize` and `serde::Deserialize`, and includes all state needed to reconstruct the `FeeTracker`: bucket statistics, block history, and current height.

---

## Specification

### FeeEstimatorState Struct

```rust
#[derive(Serialize, Deserialize)]
pub struct FeeEstimatorState {
    pub buckets: Vec<SerializedBucket>,
    pub block_history: Vec<BlockFeeData>,
    pub current_height: u64,
}
```

### SerializedBucket

```rust
#[derive(Serialize, Deserialize)]
pub struct SerializedBucket {
    pub fee_rate_lower: u128,
    pub fee_rate_upper: u128,
    pub confirmed_in_1: f64,
    pub confirmed_in_2: f64,
    pub confirmed_in_5: f64,
    pub confirmed_in_10: f64,
    pub total_observed: f64,
}
```

### Serialization Contract

- `FeeEstimatorState` MUST include all bucket data (boundaries and counters).
- `FeeEstimatorState` MUST include the full `block_history` (up to `window` entries).
- `FeeEstimatorState` MUST include `current_height` for age-based calculations.
- The serialization format MUST be compatible with `serde_json` and `bincode` (no custom serializers).

### Extraction and Restoration

- `FeeTracker::to_state() -> FeeEstimatorState` extracts the current state.
- `FeeTracker::from_state(state: FeeEstimatorState) -> FeeTracker` reconstructs from a snapshot.
- Restoration MUST produce a `FeeTracker` that returns identical `estimate_fee_rate()` results as the original at the time of snapshot.

### Integration with MempoolSnapshot

```rust
#[derive(Serialize, Deserialize)]
pub struct MempoolSnapshot {
    pub active_items: Vec<MempoolItem>,
    pub pending_items: Vec<MempoolItem>,
    pub conflict_bundles: Vec<SpendBundle>,
    pub fee_estimator_state: FeeEstimatorState,
    pub height: u64,
    pub timestamp: u64,
}
```

## Acceptance Criteria

- [ ] `FeeEstimatorState` derives `serde::Serialize` and `serde::Deserialize`.
- [ ] Includes `buckets`, `block_history`, and `current_height` fields.
- [ ] `SerializedBucket` includes all six counter/boundary fields.
- [ ] Round-trip serialization produces identical state: `from_state(to_state(tracker))` yields equivalent estimation results.
- [ ] `FeeEstimatorState` is included in `MempoolSnapshot`.
- [ ] Compatible with both JSON and binary serde formats.

---

## Implementation Notes

- **Primary file:** `src/fee/state.rs` or `src/fee/tracker.rs`
- **serde feature:** The `serde` dependency is already required for `MempoolSnapshot`. `FeeEstimatorState` uses derive macros only.
- **Versioning:** Consider a version field in `FeeEstimatorState` for forward compatibility. If bucket count or spacing changes between versions, restoration may need migration logic.
- **Edge cases:**
  - Empty state (no blocks recorded): valid, produces a tracker that returns `None` for all estimates.
  - State with different bucket count than current config: restoration should either fail gracefully or rebuild buckets.
